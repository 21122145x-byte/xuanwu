# 三、当前仍未完全闭合的部分

## 1. 每日状态池 schema 还没写完整
现在的 `daily state pool schema` 里：
- `items` 指向 `ark_item`
- 但 `definitions.ark_item` 还是空的

### 风险
这意味着：
- 单条 item schema 和每日容器 schema 还没真正接上
- 还不能直接拿这段 daily schema 做完整校验

### 当前判断
这不影响“雏形成立”，
但它阻塞下一步真正上 schema 校验。

---

## 2. 日期字段类型还偏松
当前：
- `entry_date`
- `last_review_date`
- `date`

都还是普通 string。

### 风险
实现时容易出现：
- 日期格式不统一
- 比较逻辑混乱
- 回看排序不稳定

### 最小修复
下一版建议明确：
- 日期格式（如 `YYYY-MM-DD`）
- 时间戳字段是否另存

---

## 3. 枚举值现在是英文内部值，但未给映射说明
例如：
- `main_route`
- `board`
- `validated`

这本身没问题，
但如果后面：
- 正文输出
- 审计查看
- 数据回看

也都直接看到这些内部值，
就会出现展示层和内部层混在一起。

### 当前判断
建议下一步补一张：
- 内部 enum 值
- 对外显示文案
的映射表。

这不阻塞本轮通过，
但会影响后续接前端 / 输出层。

---

## 4. 伪代码现在仍偏“逻辑顺序”，还没压到“函数契约”
当前伪代码已经很好，
但如果真要进入实现层，下一步还得补：
- 每个函数输入是什么
- 输出是什么
- 哪些字段会被更新
- 哪些字段不能被覆盖

### 当前判断
本轮不要求它写到这么细，
但下一轮如果要申请“可以开始写代码”，
这层就该补。

---
