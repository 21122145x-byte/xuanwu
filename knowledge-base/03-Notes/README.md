# 03-Notes

> 公共阶段底稿层。承载正式件之前的研究稿、分析稿、执行前准备卡与过渡性收口件。

---

## 这一层放什么
- 阶段底稿
- 研究向分析稿
- 审计前准备卡
- 执行前准备卡
- 过渡性收口件
- 尚未进入项目正式页 / Areas 正式方法页 / Constitution 正式制度页的中间材料

## 这一层不放什么
- 项目正式方案、项目入口页 → 去 `01-Projects/`
- 公共制度总纲、固定规则 → 去 `01-Constitution/`
- 纯资源资料、可查阅参考 → 去 `03-Resources/`
- 元管理索引、状态页、维护规则 → 去 `99-Meta/`
- 临时收件、未判断去向 → 先去 `00-Inbox/`

## 使用口径

### 当一份材料属于以下状态时，优先落在这里：
- 已经有分析价值，但还不是最终正式版本
- 需要给审计位、治理位、执行位先看一轮
- 是正式收口之前的准备稿或执行前卡片

### 当一份材料进入以下状态时，应考虑转出：
- 已经成为项目正式件 → 转 `01-Projects/`
- 已经抽象成公共方法论 → 转 `02-Areas/`
- 已经收为团队制度件 → 转 `01-Constitution/`
- 只剩历史留存价值 → 转 `04-Archive/`
