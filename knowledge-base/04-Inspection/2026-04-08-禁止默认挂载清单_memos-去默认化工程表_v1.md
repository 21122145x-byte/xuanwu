## 一、总原则

**必须退场的是 MemOS 的默认运行角色。**

退场对象不是“文件还在不在”，而是以下四类默认地位：
- 默认 memory slot
- 默认 auto-recall 注入链
- 默认活动 entry 地位
- 默认 allow / 默认可信基础设施地位

---

## 二、禁止默认挂载清单

| 禁止项 | 当前要求 | 原因 |
|---|---|---|
| `plugins.slots.memory -> memos-local-openclaw-plugin` | 必须解绑 | 默认记忆位绑定本体 |
| 默认 auto-recall 注入 | 必须停止 | 默认上下文污染面 |
| `plugins.entries["memos-local-openclaw-plugin"]` 活跃入口位 | 必须禁用 | 仍可能混回默认运行链 |
| `plugins.allow` 中保留 memos 默认允许地位 | 必须移除 | 仍保留默认可加载身份 |
| MemOS 作为默认正式知识入口 | 禁止 | 正式知识必须切到显式 KB |
| MemOS 作为默认可信记忆基础设施 | 禁止 | 当前主判断已否定 |
| viewer / UI 修理重新回主线 | 禁止 | 当前主线不再投 viewer 修理 |

---

## 三、固定执行顺序

必须按以下顺序执行，不允许调换：

1. **slot unbind**
2. **entry disable**
3. **allow remove**

说明：
- 如果先删 allow，但 slot 仍在，默认角色并未真正退出
- 如果先禁 entry，但 auto-recall 或 slot 逻辑还活着，也不算退场完成

---

## 五、验收判定（禁止默认挂载侧）

以下 6 条必须全部满足，才算默认挂载真正退出：

1. 默认 memory slot 不再指向 MemOS
2. 默认 auto-recall 注入已停止
3. memos plugin 不再处于活动 entry 态
4. memos plugin 不再拥有默认 allow 身份
5. 正式知识入口已切到文件层 / KB
6. 文档口径中不再把 MemOS 写成默认可信基础设施

---

## 六、一句话收口

> MemOS 可以留材料，不可以留默认角色；任何以“研究保留”为名延长默认挂载的做法，当前都不成立。
