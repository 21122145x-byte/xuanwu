## 一、当前审计口径

本次审计对象已经明确为：

> **MemOS `1.0.7 → 1.0.8` 升级完成后的升级后安全扫描。**

因此本稿不再回答：
- 当前到底是不是 `1.0.3`
- 当前到底是不是 `1.0.7`
- 旧版本识别链谁对谁错

本稿现在只回答：
1. 升级后当前系统暴露面如何
2. 升级后是否消除了旧风险
3. 升级后是否引入新漂移或新冲突
4. 升级后是否改变“去 memos 化”主判断

## 三、升级后已成立的判断

### 1. 升级没有自动收紧风险暴露面
当前升级到 `1.0.8` 后，系统仍然存在高风险暴露面，且主要风险并不来自版本号本身，而来自当前运行配置组合：
- `groupPolicy = "open"`
- `exec.security = full`
- `sandbox = off`
- `memos-local` 仍挂在默认 `memory slot`

因此当前可以成立：

> **`1.0.7 → 1.0.8` 升级没有自然带来安全收敛。**

### 2. MemOS 仍然是默认 memory provider
当前配置仍然是：
- `plugins.slots.memory = memos-local-openclaw-plugin`
- plugin 已初始化
- auto-recall 仍然启用

因此当前可以成立：

> **升级后 MemOS 仍在默认记忆位，去 memos 化判断没有被推翻。**

### 3. viewer / service 启动问题没有自动消失
升级后日志直接显示：
- host 未调 `service.start()`
- plugin 自行尝试拉起 viewer
- viewer 连续端口冲突后失败

因此当前可以成立：

> **升级到 `1.0.8` 后，viewer 启动与端口占用问题仍然存在。**

### 4. 插件代码安全红旗仍在
安全审计继续报：
- `plugins.code_safety`
- `memos-local-openclaw-plugin contains dangerous code patterns`

说明：
> **升级到 `1.0.8` 并未改变该插件在代码安全审计中的高风险属性。**

## 四、当前仍未成立的判断

### 1. 不能写成“1.0.8 更安全”
当前没有证据支持：
- `1.0.8` 已明显降低当前环境风险
- `1.0.8` 已显著改善暴露面

### 2. 不能写成“旧风险已修复”
当前仍不能成立：
- auto-recall 边界污染已解决
- viewer 问题已解决
- gateway probe / memory unavailable 已解决

### 3. 不能写成“升级导致新增回归”
当前虽然升级后问题仍在，
但在没有明确 before/after 对照前，也不能直接写成：
- `1.0.8` 新增了这些问题

更稳的口径只能是：
> **升级后未见旧风险自然消退，且运行健康面仍不理想。**

---

## 五、当前最危险的风险组合

### P0 组合风险
#### 1. open group policy
- Discord 全局与各账号：`groupPolicy = open`
- Feishu 全局与各账号：`groupPolicy = open`

#### 2. exec full
- `anna` 仍为 `exec.security = full`

#### 3. sandbox off
- `agents.defaults.sandbox.mode = off`
- 多 agent 暴露 runtime / fs 能力

#### 4. memos 仍为默认记忆位
- `plugins.slots.memory = memos-local-openclaw-plugin`
- `allowPromptInjection=true`
- `auto-recall enabled`

一句话：
> **升级后的最高风险，仍然是“开放群策略 + 高权限工具 + 无沙箱 + 默认挂载 memos”四件事叠在一起。**

这说明：
> **升级后插件版本元数据记录未同步收口，后续会继续制造口径混乱。**

### 2. Gateway probe failed
当前仍报：
- `gateway.probe_failed`

因此：
> **升级后运行探测面仍不健康。**

### 3. Memory 状态仍 unavailable
`openclaw status` 里仍显示：
- `Memory: enabled ... unavailable`

因此：
> **升级后不能把“插件已初始化”等同于“记忆链已健康可用”。**

## 七、当前修复优先级建议

### P0
1. **收紧 Discord / Feishu 的 `groupPolicy`**
   - 从 `open` 收回到 `allowlist` 或更窄范围
2. **隔离 `anna` 的 `exec.security=full`**
   - 不再让其暴露在 open 群策略下

### P1
3. **继续推进去 memos 化**
   - 解除 `plugins.slots.memory = memos-local-openclaw-plugin`
   - 不再让 MemOS 挂默认位
4. **修复插件安装记录漂移**
   - 避免后续继续口径混乱

### P2
5. **排查 viewer 端口冲突与 memory unavailable**
6. **排查 gateway probe timeout**

## 八、当前正式结论

> **MemOS 从 `1.0.7` 升级到 `1.0.8` 后，当前最关键的结论不是“版本是否更先进”，而是：升级并未自然消除系统当前的主要安全与运行风险。**

更准确地说：
- 升级后 MemOS 仍然挂在默认 memory slot
- open group policy + exec/full + sandbox off 的高风险组合仍然成立
- viewer / memory unavailable / gateway probe 问题仍在
- 插件代码安全红旗仍在

因此：
> **本次升级后扫描支持“继续推进去 memos 化与暴露面收紧”，不支持“升级已改善安全态”这一说法。**

## 十、下一步建议

### 1. 先收紧开放暴露面，不再继续放任当前组合运行
当前最先该做的不是继续讨论 MemOS 版本，而是先收紧这组高风险组合：
- `groupPolicy = "open"`
- `exec.security = full`
- `sandbox = off`
- `memos-local` 仍在默认 `memory slot`

建议顺序：
1. 先把 Discord / Feishu 的 `groupPolicy` 从 `open` 收回到 `allowlist` 或更窄范围
2. 把 `anna` 的 `exec.security=full` 从开放群可达范围中隔离出去
3. 再决定是否继续保留当前默认 memory slot 绑定

### 2. 继续推进去 memos 化，不再把版本升级当成主解法
本次 `1.0.7 → 1.0.8` 升级后，当前已足够说明：
> 版本升级不会自然解决当前的 recall 污染、viewer 启动、默认挂载与攻击面暴露问题。

因此建议继续推进：
- 解除 `plugins.slots.memory = memos-local-openclaw-plugin`
- 把本地替代层作为正式主线
- 将 MemOS 从“默认基础设施”降级为“待迁出旧依赖”

### 3. 把安装记录漂移列为必须补的台账修正项
当前审计已报：
- `plugins.installs_version_drift`
- `recorded 1.0.7, installed 1.0.8`

建议把它列为：
> **必须补的版本台账修正项。**

原因：
- 它已经造成过一次事实判断错误
- 后续如果不修，会继续污染审计口径和升级判断

### 4. viewer / gateway / memory unavailable 进入并行排查，但不抢 P0 主线
当前仍存在：
- viewer 端口冲突与自启动失败
- gateway probe timeout
- `Memory: unavailable`

这些都应进入下一轮并行排查，
但优先级应排在：
1. 暴露面收紧
2. 去 memos 化
之后。

### 5. 升级后复测结果要写成 before/after，而不是口头印象
如果后续还要继续评估 MemOS 是否保留某些能力，建议补一页简短 before/after：
- 升级前：1.0.7
- 升级后：1.0.8
- 变化项：
  - 是否减少报错
  - 是否改善 viewer
  - 是否改善 memory availability
  - 是否改善 auto-recall 边界

在没有 before/after 前，不建议把这次升级写成：
- 新增回归
- 安全改善
- 运行改善

### 6. 当前正式建议
如果把下一步压成一句最短行动建议：

> **先收紧开放策略，再继续去 memos 化，同时把 1.0.8 的安装记录漂移修正掉；viewer / gateway / memory unavailable 作为下一轮并行问题收证，不再把版本升级本身当成主解法。**
