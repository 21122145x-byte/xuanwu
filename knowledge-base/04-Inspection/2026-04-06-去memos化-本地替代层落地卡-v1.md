# 去 memos 化｜本地替代层落地卡 v1

- 日期：2026-04-06
- 主责：Anna
- 性质：执行卡

---

## 一、目标

这张卡只回答一件事：

> **memos 退场后，本地记忆、知识、共享、检索由什么来承接。**

当前不追求“替代一个插件”，
而是追求：

> **把默认承载从隐式注入记忆，改成显式文件记忆 + 显式知识库 + 可控检索工具。**

---

## 二、本地替代层结构

## 层 A：工作记忆层
### 载体
- `/Users/xiaojingbo/.openclaw/workspace-anna/MEMORY.md`
- `/Users/xiaojingbo/.openclaw/workspace-anna/memory/YYYY-MM-DD.md`

### 职责
- 长期记忆摘要
- 当日推进记录
- 承诺 / 阻塞 / 关键决策留痕
- 不依赖自动 recall，靠显式写入与显式读取

### 使用原则
- 长期项进 `MEMORY.md`
- 当日项进 `memory/YYYY-MM-DD.md`
- 不把系统噪音、注入文本、工具壳内容写进工作记忆层

---

## 层 B：显式知识层
### 载体
- `/Users/xiaojingbo/.openclaw/workspace-anna/knowledge_base/`

### 职责
- 承接正式文档
- 承接审计意见 / RCA / 执行卡 / 规则 / SOP
- 承接索引、topic、reports、design、todos

### 使用原则
- 正式结论优先落 knowledge_base
- 不再把 memos 当正式知识承载层
- 有长期价值的 memos 维修经验，改挂到显式知识层

---

## 层 C：本地可控检索层
### 载体
- `memory_search`
- `memory_get`
- `memory_timeline`
- `memory_write_public`
- `memory_share`

### 职责
- 作为工具化检索层使用
- 仅在需要时主动查，不做默认前置注入
- 承接有限共享记忆，而非默认上下文污染

### 使用原则
- 检索应手动触发 / 场景触发
- 不承担默认 prependContext 职责
- 共享前先判断是否真有跨 agent 复用价值

---

## 层 D：技能与规则层
### 载体
- 本地 skills
- 显式规则文档
- 审计 skill / 执行 skill / wiki-maintenance 类技能

### 职责
- 让“经验”以 skill / rule 的形式复用
- 不再依赖 memos 插件把经验偷偷塞回上下文

### 使用原则
- 经验进入 skill 或显式知识文件
- 技能说明、references、scripts 走文件化承载
- 不把治理经验继续绑在 memos 插件上

---

## 三、退场后最小工作闭环

memos 退场后，Anna 默认工作闭环改为：

1. **启动时显式读**
   - `SOUL.md`
   - `USER.md`
   - `MEMORY.md`
   - `memory/YYYY-MM-DD.md`

2. **需要知识时显式查**
   - 先读 `knowledge_base` 索引/文档
   - 再按需用 `memory_search`

3. **有结论时显式落文**
   - 审计 / 建议 / RCA / 规则 → `knowledge_base`
   - 个人/当日状态 → `memory/`

4. **需要共享时显式写 / share**
   - 用 `memory_write_public` 或正式交接
   - 不靠 auto-recall 被动扩散

---

## 四、第一轮最小落地顺序

### 动作 1：确认本地文件记忆层为默认主层
统一口径：
- `MEMORY.md` + `memory/YYYY-MM-DD.md` = 默认记忆主层
- knowledge_base = 默认正式知识主层

### 动作 2：停止把 memos 当默认知识入口
统一口径：
- 不再引用 memos viewer 作为主入口
- 不再把 memos recall 当默认背景层

### 动作 3：将有价值的 memos 经验显式回挂
优先回挂到：
- `knowledge_base/10-CURRENT-STATUS/`
- `knowledge_base/20-DESIGN/`
- `knowledge_base/30-RETROS/`
- `knowledge_base/50-REPORTS/`

### 动作 4：共享记忆改为显式写入
- 局部共享：`memory_write_public`
- 团队共享：正式 handoff / team KB

---

## 五、当前验收标准

替代层成立，只看这 4 条：
1. 不依赖 memos auto-recall，也能完成默认启动与工作
2. 正式知识有显式落点，不再靠隐式记忆承载
3. 需要检索时可用手动 memory 工具补位
4. 共享动作变成显式写入，而不是被动注入
