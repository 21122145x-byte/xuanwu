## 一、当前使用原则

### 1. 总入口只有一个
治理体系总入口为：
- `README.md`

它负责回答：
- 当前目录怎么读  
- 五层主链顺序是什么  
- 哪些文件是正式入口  
- 哪些文件不应默认先读

### 2. 五层主链各有稳定入口
当前五层正式入口为：

1. `团队协作宪法与治理总纲.md`  
2. `Program-团队协作运行主协议.md`  
3. `Project-Plan规范说明及模板卡.md`  
4. `SOP-Runbook-Checklist规范说明及模板卡.md`  
5. `Prompt规范说明及模板卡.md`

### 3. Program 线只保留两张核心附件
- `任务级Program文件规范卡.md`  
- `Program与runtime-pipeline接口卡.md`

默认不要把附件当成主链正文，也不要默认主文 + 全附件组并读。

## 二、默认阅读顺序

若从头进入治理体系，默认按以下顺序阅读：

### Step 1：先读 Charter
- `团队协作宪法与治理总纲.md`

### Step 2：再读 Program
- `Program-团队协作运行主协议.md`

### Step 3：再读 Project Plan
- `Project-Plan规范说明及模板卡.md`

### Step 4：再读执行层
- `SOP-Runbook-Checklist规范说明及模板卡.md`

### Step 5：最后读 Prompt
- `Prompt规范说明及模板卡.md`

### Step 6：仅在需要时加读 Program 附件
- `任务级Program文件规范卡.md`  
- `Program与runtime-pipeline接口卡.md`

---

## 三、如果你已经从某一层进入

### 1. 从 Charter 进入
读完后默认继续：
- `Program-团队协作运行主协议.md`

### 2. 从 Program 进入
读完后默认继续：
- `Project-Plan规范说明及模板卡.md`

必要时才加读：
- `任务级Program文件规范卡.md`
- `Program与runtime-pipeline接口卡.md`

### 3. 从 Project Plan 进入
读完后默认继续：
- `SOP-Runbook-Checklist规范说明及模板卡.md`
- `Prompt规范说明及模板卡.md`

### 4. 从 Prompt 进入
Prompt 以下不再继续分治理层。  
若边界不清，应向上回看：
- `Project-Plan规范说明及模板卡.md`
- `Program-团队协作运行主协议.md`
- `团队协作宪法与治理总纲.md`

## 五、子文件夹说明

### `01-版本实体/`
当前稳定入口所承接的版本实体 / 内容源。

### `02-历史版本/`
历史版本件与历史迁出版，只用于审计、diff、回滚与复盘。

### `03-审计与收口/`
审计裁决、任务单、回执、总说明、整理记录等。

### `04-结构与导航素材/`
索引卡、分工卡、法位表、规则卡、结构建议等。

### `05-术语与过渡件/`
术语澄清件与术语去向整理说明。

### `06-实例与迁移/`
项目实例件与迁移辅助件。

## 六、默认不要先读什么

除非当前任务明确要求，否则默认不要先读：

- `01-版本实体/` 中的版本实体件  
- `02-历史版本/` 中的历史版本件  
- `03-审计与收口/` 中的审计与回执件  
- `05-术语与过渡件/` 中的术语澄清件  
- `06-实例与迁移/` 中的实例件与迁移件

这些文件有价值，但不属于默认最小阅读链路。

进入《团队治理体系》时，默认这样读：
