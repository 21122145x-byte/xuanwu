## 一、当前目录分组原则

当前目录中的文件，统一按以下法位分组理解：

1. **正式主链入口件**：当前正式唯一入口  
2. **当前版本主体件**：当前入口所承接的版本实体 / 内容源  
3. **历史版本件**：保留用于审计、diff、回滚、复盘的历史版本  
4. **迁出版**：为解决入口语义冲突而迁出的旧内容件  
5. **正式规则 / 审计件**：用于规则确立、裁决与返工判断的正式辅助件  
6. **正式附件**：正式依附于某一主链层级、承担稳定补充规则的附件  
7. **说明附件**：说明、术语、接口、结构理解辅助件  
8. **实例 / 迁移辅助件**：项目实例、迁移表、建设过程辅助件  
9. **总说明 / 交接件**：面向交接、审计入口与阶段性状态说明的汇总件

## 十二、当前目录阅读顺序建议

若首次进入当前目录，建议按以下顺序阅读：

### 第一层：先看正式主链入口件
- Charter → Program → Project Plan → SOP / Runbook / Checklist → Prompt

### 第二层：再看规则 / 审计件
- 唯一入口与版本管理规则卡  
- 首轮审计裁决表  
- 二轮补充裁决表

### 第三层：再看 Program 附件与说明附件
- 任务级 Program 文件规范卡  
- Program 与 runtime pipeline 接口卡  
- Skill / MCP / Harness / Runtime Pipeline 关系卡

### 第四层：最后看历史版本件、实例件与迁移件
- 各类 v0.x / v2.x 历史版本  
- 迁出版  
- 项目实例
