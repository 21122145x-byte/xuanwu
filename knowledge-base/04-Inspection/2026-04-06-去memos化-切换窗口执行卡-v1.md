# 去 memos 化｜切换窗口执行卡 v1

- 日期：2026-04-06
- 主责：Anna
- 性质：执行卡

---

## 一、切换目标

这次切换只做一件事：

> **解除 memos 作为默认 memory slot / 默认上下文记忆基础设施的绑定。**

当前不做：
- 物理删除所有 memos 文件
- 立刻销毁历史数据库
- 一次性清空所有相关知识文档

当前只做：
- 冻结默认角色
- 解除配置绑定
- 保留历史资产
- 验证本地替代层是否接住主闭环

---

## 二、切换前冻结项

## 冻结 1：停止把 memos 当默认可信记忆层
统一口径改为：
- memos = 待迁出旧依赖
- 非默认记忆真相层
- 非默认上下文注入层

## 冻结 2：停止继续围绕 viewer / auto-recall 做主线修理
后续不再以修 viewer / auto-recall 为优先主线。

## 冻结 3：冻结历史数据路径
保留但不主动依赖：
- `~/.openclaw/memos-local/`
- `~/.openclaw/memos.db`
- backup / hold 目录

---

## 三、配置层动作

## 动作 1：解除 memory slot 绑定
目标：
- 将 `plugins.slots.memory` 从 `memos-local-openclaw-plugin` 解绑

## 动作 2：禁用 memos plugin entry
目标：
- `plugins.entries["memos-local-openclaw-plugin"].enabled = false`

## 动作 3：从 `plugins.allow` 中移除 memos-local-openclaw-plugin
目标：
- 不再允许其作为默认活动插件参与加载

### 当前原则
配置切换顺序必须是：
1. 先解绑 slot
2. 再禁用 entry
3. 最后从 allow 移除

避免留下“slot 仍指向已禁用插件”的悬空状态。

---

## 四、物理层动作边界

## 当前允许
- 保留插件目录
- 保留数据库文件
- 保留备份目录
- 保留调试日志

## 当前不允许
- 直接删除 `~/.openclaw/extensions/memos-local-openclaw-plugin`
- 直接删除 `~/.openclaw/memos-local/memos.db`
- 直接清空 backup / hold

### 原因
当前仍处于迁移窗口，
应先完成：
- 配置退场
- 工作闭环验证
- 历史资产分类

然后再决定是否物理归档。

---

## 五、知识与技能资产处理

## 1. 继续保留，但改口径
当前所有 memos 相关文档：
- 不删除
- 不再挂“当前默认基础设施”口径
- 改挂为：历史问题簇 / 治理经验 / 架构演化资产

## 2. memos 相关 skill 进入迁移池
包括：
- `memos-memory-guide`
- skills-store 下相关 memos skill

处理原则：
- 保留经验
- 去插件依赖
- 能迁入本地知识与新 skill 结构的，逐步迁入

## 六、切换后验收步骤

切换完成后，至少验证：

### 验收 1：默认工作启动不再依赖 memos
- 启动读取仍走：`SOUL.md / USER.md / MEMORY.md / memory/YYYY-MM-DD.md`
- 无 memos 也能完成默认工作闭环

### 验收 2：不再发生 memos auto-recall 注入
- 主会话默认不再出现来自 memos 的 prependContext 记忆注入

### 验收 3：正式知识仍可通过 knowledge_base 读取
- 索引、topic、report、design 文件仍可正常承接知识

### 验收 4：可控检索仍可由本地工具补位
- 必要时用 `memory_search` / `memory_get` 做显式检索
- 不依赖自动 recall

---

## 七、回滚边界

## 允许回滚的条件
只有在以下情况同时出现时，才考虑短期回滚配置：
- 替代层未接住基本工作闭环
- 显式知识 / 文件记忆 / 本地工具层无法支撑默认工作
- 且短期内无其他替代方案

## 当前不允许的回滚理由
以下都不构成回滚理由：
- “viewer 也许还能修”
- “历史上它曾经可用”
- “自动注入比较省事”

---

## 八、不做事项

这次窗口明确不做：
- 重写宿主 memory architecture
- 新建复杂 memory backend
- 立刻整体替换为另一套同类 auto-recall 插件
- 在无验收情况下启用新的默认自动记忆注入机制

---

## 九、一句话收口

> **这次切换窗口的目标不是“销毁 memos”，而是“解除它的默认绑定并让本地替代层接管默认工作闭环”；先退场，再冻结，再验证，最后才决定是否物理归档。**
