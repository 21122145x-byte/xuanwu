# 半自动任务阶段门状态模板 v1

- 适用协议：`/Users/xiaojingbo/.openclaw/workspace/knowledge-base/04-Inspection/2026-04-11-半自动任务执行协议_v1.md`
- 用途：统一记录半自动任务当前阶段、责任位、下一动作、停滞门位、流转状态与阻塞信息

```md
# Task Status Card

Task: <任务名>
status: <drafting | revision-needed | review-ready | sent-to-audit | audit-in-progress | audit-blocked | awaiting-human-confirmation | blocked | accepted | returned | announced | closed>
owner: <当前责任位>
next_owner: <下一责任位>
next_action: <下一动作>
updated_at: <YYYY-MM-DD HH:mm Asia/Shanghai>
stalled_at_gate: <如无可留空；如有则写 review-ready / sent-to-audit / audit-in-progress / other>
flow_state: <flowing | not-flowing>
alive_signal: <当前链路仍 alive 的信号，例如：已收到交付 / worker 仍在运行 / 审计位已回执>
flow_signal: <当前阶段是否继续向下一门推进的信号，例如：已送审 / 已开始审计 / 已转返修>
artifact_where:
- <主文路径>
- <附件/支持材料路径>
known_issues:
- <当前已知问题>
decision_basis:
- <为什么进入当前状态>
handoff_mode: <main-agent-send | session-send-reminder | document-reference | message-notify | internal-mark>
blocking_reason: <如无可留空>

## Notes
- <补充说明>
```

## 字段说明补充
- `stalled_at_gate`：用于显式标记“链路 alive，但阶段未继续流转”时，当前卡住的是哪一道门。
- `flow_state`：
  - `flowing` = 当前阶段仍在真实推进
  - `not-flowing` = 链路可能还活着，但状态没有继续向下一门流转
- `alive_signal`：回答“这条链是不是还活着”。
- `flow_signal`：回答“这条链是不是还在继续流转”。
- 若 `status` 长时间未变化且业务链未向下一阶段推进，应优先补 `stalled_at_gate` 与 `flow_state`，不要只写泛化 blocker。
- `stable review-ready` 只代表当前观察点稳定，不等于业务链已经自然闭合。
