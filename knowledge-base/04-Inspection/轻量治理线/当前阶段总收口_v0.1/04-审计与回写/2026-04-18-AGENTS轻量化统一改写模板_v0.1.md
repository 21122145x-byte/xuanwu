## 2. 执行默认规则（统一）
- 默认按版本卡执行，不再各自临场发明起手顺序
- 执行原则：**一个最小单元 → 一个 prompt → 一个 subagent → 结束就删**
- 执行中如需 keeper / 审计 / 断点恢复 / 复盘 / 回挂，按版本卡后续步骤走

## 3. Discord 跨 Agent 交接规则（统一）

最小写法应直接挂在 `AGENTS.md`：

## 4. 知识库文档管理规则（统一）
默认写法应直接挂在 `AGENTS.md`：
- 正式结论 / SOP / 主文 / 规则卡 / 交接文档 → 进入 `knowledge-base` / `knowledge_base`
- 临时聊天结论不算正式沉淀

- 能挂主文的不只停在聊天里
- 历史阶段文保留为历史，不与当前主体混位

## 6. 改写原则
- 轻量化，不写成长说明书
- 共法放上位文，不在每个 agent 文件里重复展开
- 每个 agent 只保留：专属职责、专属边界、必要接口
- 团队统一动作统一写法，不各写一套变体
