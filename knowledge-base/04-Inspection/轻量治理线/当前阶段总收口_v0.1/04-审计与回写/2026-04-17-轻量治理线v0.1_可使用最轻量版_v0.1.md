## 三、当前默认怎么用

### 1. `minimal-unit-builder`
#### 可直接轻量使用
用于：
- 厚任务压薄
- 最小可执行对象判断
- 伪缩小识别

#### 当前不这么用
- 不让它重写整条主线
- 不让它替主 Agent 接管任务改写

---

### 2. `main-artifact-judge-helper`
#### 可轻量使用，但必须带红线
用于：
- 当前阶段主产物判断面压清
- 主产物 vs 配件 / 桥接件 / 入口卡 / 说明件区分
- 输出“当前判断 + 依据 + 待验证点”

#### 当前不这么用
- 不把它当最终裁决器
- 不让它替主 Agent 做最终项目归位
- 不让它靠“最显眼 / 最完整 / 最方便接链”做捷径判断

---

### 3. `revision-needed-router`
#### 可轻量使用，但必须带复杂度红线
用于：
- `revision-needed` 后回哪层
- 补件后回哪个最近有效判断面
- accepted 后接哪一步
- 输出“建议 + 依据 + 待确认点”

#### 当前不这么用
- 不写成自动上提器
- 不把复杂多分叉回退强压成单线
- 不吸收主产物判断 / 注册层入口判断 / 组级阶段判断

## 七、复盘这块在 v0.1 里怎么承接

### 当前只承接一件事
> **复盘的默认动作。**

也就是：
- 当需要复盘沉淀时，允许触发复盘默认动作
- 但不把完整复盘机制链搬进 v0.1 主线

### 当前不并入 v0.1 主线的内容
以下完整后续链，当前都不并入：
- 来源抓取 / 转写
- 对象抽取
- 状态判断
- 周升级
- 上提为可复用资产
- 回写到入口 / 索引 / 收口件

原因很简单：
1. 当前还没写清楚
2. 一并进来会把 v0.1 做厚
3. 不符合“可使用最轻量版”口径

### 复盘结束后的最轻量收口动作
当前只收在这三步：
1. **整理产物**
2. **回写计划表**
3. **补最小导航**

这就是 v0.1 当前对复盘的最轻量承接。

## 八、当前明确不做什么

v0.1 当前明确**不追求**：

1. 不做完整制度化
2. 不做样本充分闭合
3. 不做所有机制一起 skill 化
4. 不做厚重复盘整理制度并入
5. 不做一整套来源抓取 / 周升级 / 资产上提链
6. 不把“可用最轻量版”夸大成“最终发布完成版”

也就是说：
> **现在先保证能用、顺、薄；不先追求全。**

## 十、当前最稳执行原则

1. **先用，不先扩机制**
2. **先清入口和顺序，不先补厚规则**
3. **先当可用助手，不当自动裁决器**
4. **使用本身就是升级入口**
5. **边用边补样本，不先堆制度完整版**

## 十一、当前下一步怎么推进

如果继续顺推 v0.1，当前最合理的动作是：
1. 保持主入口和默认顺序稳定
2. 在真实使用中调用这三张最轻量版正式 skill
3. 遇到新增误判面时，记为样本补强
4. 必要时再做轻量回写，不重新拉起厚机制
