- 日期：2026-04-16
- 起草：粉雪
- 状态：completed
- 性质：口径卡 / 当前版本目录治理件
- 用途：统一《轻量治理线 / 当前阶段总收口_v0.1》目录内关键文档的头部元信息口径，降低“目录结构已分层，但文档自述仍无层级信息”的混读风险。
- 版本目录：轻量治理线 / 当前阶段总收口_v0.1
- 所属层级：审计与回写
- 当前阶段：项目化收纳 → 分层收纳 → 导航收口后的元信息统一

---

## 一、当前统一字段（最小集）
当前版本目录中的关键文档，头部优先统一以下字段：
- 日期
- 起草
- 状态
- 性质
- 用途
- 版本目录
- 所属层级
- 当前阶段

## 二、字段解释
### 1. 版本目录
用于说明该文档当前被收纳在哪个项目版本目录中。
默认写法：
- `轻量治理线 / 当前阶段总收口_v0.1`

### 2. 所属层级
用于说明该文档当前在项目目录中的归位层级。
推荐值示例：
- 版本导航
- 主方案与主文
- 注册层 / 总览与清单
- 注册层 / 规则表件
- 注册层 / 纠偏与收口
- 候选 skill / 总览与编排
- 候选 skill / 三张主焦点
- 候选 skill / 其余候选链
- 审计与回写
- 历史参考与 RCA

### 3. 当前阶段
用于说明该文档此刻处于哪一段项目阶段，不代替文档自己的内容状态。
本轮推荐写法：
- `当前阶段总收口`
- `真实样例试跑前收口`
- `提交审计 / revision-needed 轻修期`
- `项目化收纳 / 分层收纳 / 导航收口期`

## 三、本轮 적용边界
本轮只统一：
- 版本导航
- 各层入口 README
- 注册层默认起手核心件
- 候选推进总览件
- 提交审计件
- 审计意见件

本轮不做：
- 全目录 90+ 文件一次性全量改头
- 旧历史件逐张重写
- 文档正文结构大改

## 四、一句话口径
> 当前先让“默认会被打开的文档”头部自述和目录层级对齐；其余历史件后续再逐层清理。
