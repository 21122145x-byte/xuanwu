## 一、任务类型读取表

| 任务类型 | 默认先读 | 再读 | 最后才回看 |
|---|---|---|---|
| 项目态续接 | 项目总表 | 主文 → 当前阶段规则件 | README / 聊天 |
| 编排任务 | 项目总表 | 主文 → 最小单元 / 阶段门相关件 | 缺口表 / 聊天 |
| 审后回转任务 | 上一轮审计结论 | 项目总表 → 阶段门 / 回转相关件 | 聊天 |
| 复盘任务 | 复盘件 | 项目总表 → 对应问题件 / 规则件 | 聊天 |
| 停损 / 升级任务 | 停损规则卡 | 项目总表 → 当前阶段判断 | 聊天 |
| 导航修订任务 | README | 主文头部 → 项目总表 | 聊天 |
| 注册层补件任务 | 注册层整理表 | 现役能力清单 → 映射表 → 相关候选链 | README |
| 候选 skill 推进任务 | 机制与skill映射表 | 候选拆解卡 → 说明草案 → 样例集 → 审计记录 | 聊天 |

---

## 二、使用提醒

- 不同任务类型默认入口不同，不能一律从聊天开工
- 导航修订任务可先读 README，但项目态任务不行
- 审后回转任务必须先回审计结论，不能只看当前对话

---

## 三、一句话收口

> 按任务类型读取表补上后，注册层不再只会回答“总体先读谁”，而开始能回答“这类任务具体先读哪一层”。
