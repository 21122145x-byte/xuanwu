## 一、总优先级

默认裁决优先级：
1. 项目总表
2. 主文
3. 对应规则卡 / 模板件
4. 已收口的新件
5. README / 展开表 / 缺口表
6. 聊天记录 / 旧版本文档

---

## 二、常见冲突场景

| 冲突场景 | 默认先信谁 | 说明 |
|---|---|---|
| 项目总表 vs 主文 | 项目总表（阶段定位） / 主文（机制定义） | 阶段归属先信总表，机制定义先信主文 |
| 主文 vs 规则卡 | 主文（主线定义） / 规则卡（具体执行） | 规则卡不能反向重写主文主线 |
| 规则卡 vs 模板件 | 规则卡 | 模板件服务规则，不反向裁决规则 |
| 新件 vs 旧件 | 已收口的新件 | 新件优先，但前提是已明确挂回主链 |
| README vs 正式文档 | 正式文档 | README 仅导航，不裁决 |
| 聊天记录 vs 正式文档 | 正式文档 | 聊天不可反向定义当前口径 |

---

## 三、边界

本表只处理默认裁决优先级，不处理：
- 高风险例外裁决
- 审计放行裁决
- 停损裁决

这些仍应回专门规则件。
