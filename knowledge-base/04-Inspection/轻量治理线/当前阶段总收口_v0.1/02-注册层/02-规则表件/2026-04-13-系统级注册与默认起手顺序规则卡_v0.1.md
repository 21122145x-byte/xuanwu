## 1. 规则目标
本规则只承接三件事：
1. agent 默认如何起手
2. 系统级注册层如何读
3. 项目态任务为什么必须先回项目总表

## 2. 默认起手顺序
当任务进入项目态或连续推进态时，主入口 Agent 默认按以下顺序起手：
1. 先确认任务目标与当前阶段
2. 再读系统级注册层
3. 再看当前项目总表
4. 再决定当前默认下一动作
5. 最后才决定是否进入派单或生产

一句话口径：
> 项目态任务先回注册层与项目总表，不先靠聊天记忆直接开工。

## 3. 系统级注册层最少要读什么
主入口 Agent 至少应判断：
- 当前有哪些现役能力 / skill / 模板 / 规则卡
- 默认先读谁
- 冲突时先调谁
- 哪些入口已废弃或仅供参考

## 4. 项目总表的默认作用
项目总表在项目态任务中的默认作用是：
- 记录当前阶段
- 记录已完成项
- 记录当前关键缺口
- 记录默认下一步
- 记录若当前不推进时应回写的状态

## 5. 默认禁忌
以下做法不应作为项目态任务的默认起手：
- 只凭聊天上下文决定下一步
- 不读注册层就直接开工
- 不看项目总表就临场决定派单方向
- 用后续计划表或甘特图替代项目总表作为默认入口

## 6. 当前结论
这张规则卡的作用不是新增一套大机制，而是把主文中“系统级注册 / 默认起手顺序 / 项目总表入口”这层补成一个可挂接的承接件。
