## 1. 规则目标
当协作继续按当前方式推进，新增成本已明显高于新增收益时，停止硬拖与假推进，进入正式停损判断。

## 2. 停损触发信号
出现以下任一类信号时，应进入停损判断：
- 主产物连续不增长
- 小单元错误开始牵连整阶段返工
- 回收成本显著高于继续生产收益
- 审计连续指出同类结构问题
- runtime keeper 连续发出卡断 / timeout / 假推进提醒
- 预算消耗开始偏离主线目标
- 任务目标已变化，但计划未重编排

## 3. 谁能提停
- 主入口 Agent：可正式提出停损
- runtime keeper：可发出停损提醒
- 审计位：可提出停损建议
- 人类拍板位：可做最终停损裁定

## 4. 停损后的默认路径
停损成立后，默认只走以下四条路径之一：
- 回本阶段重编排
- 缩小粒度后恢复推进
- 暂停并升级
- 阶段性收口后转复盘

## 5. 执行口径
- 停损是正式节点，不是口头犹豫
- 停损优先于假推进
- 恢复优先于硬拖
- 停损后必须明确去向，不允许停在悬空状态

## 6. 最小回收标准
本卡达到“可回收、可继续审”的最小状态，至少应满足：
1. 已明确停损触发信号
2. 已明确谁能提停
3. 已明确停后默认分流路径
4. 未扩写成完整治理手册
5. 未混入审计插入规则正文
