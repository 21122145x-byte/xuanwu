## 二、下一步优先顺序（Anna 口径）

当前后续轮转优先顺序固定为：

### A. 审后回转试跑
**优先级：P1 / 第一优先**

目标：
- 验证 `accepted / revision-needed` 出来后，主入口位如何继续接链
- 验证是否能进入下一阶段，或回本阶段重编排

为什么优先：
- 当前缺口不在送审前，而在送审后
- 这是让闭环“再转一圈”的最短补口路径

### C. runtime keeper 介入试跑
**优先级：P2 / 第二优先**

目标：
- 验证真实卡断 / timeout / 假推进场景下，runtime keeper 如何介入

为什么排第二：
- keeper 目前已定义，但尚未进入真实运行态证据层
- 适合在审后回转链补上之后继续验证

### B. blocked 升级试跑
**优先级：P3 / 第三优先**

目标：
- 验证 blocked 结论出现后，如何升级到更重分支

为什么排第三：
- blocked 属于更重场景
- 当前更高频、更基础的是先跑通审后回转

## 三、当前禁止误判

当前不应误判为：
- 项目已经完成
- 机制已全链闭合
- 可以不做 `A > C > B`
- 下一步可以随意跳题

当前正式口径应是：

> 项目已完成主文与送审前最小闭环试跑；下一步按 `A > C > B` 推进，未完成前不视为机制轮转链成立。

---

## 四、如果当前不立即做 A > C > B，下一步该做什么

若当前不立即继续试跑，不代表“没有下一步”，而是应先做以下治理动作之一：

1. **先维护这张计划表**
   - 把新审计结论、新试跑结果、新优先级变化回写到这里
2. **以这张计划表作为唯一项目推进入口**
   - 后续不再临场重新讨论“下一步做什么”
3. **后续任何试跑都先挂回本表**
   - 先改状态，再开跑

一句话规则：

> 如果不立刻做 `A > C > B`，那当前唯一正确动作就是维护计划表，而不是临场凭聊天重新决定项目方向。
