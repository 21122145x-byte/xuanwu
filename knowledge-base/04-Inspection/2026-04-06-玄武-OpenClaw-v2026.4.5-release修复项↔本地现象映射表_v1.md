- 日期：2026-04-06
- 归属：玄武
- 类型：版本映射审计表
- 上游版本：`openclaw/openclaw@v2026.4.5`
- 用途：将 release 修复项与当前本地已观测现象逐条建立“相关性而非定因”的映射，用于后续审计与 before/after 验证设计。

---

## 1. 使用边界

这张表当前**可以做**的事：
- 作为版本风险背景输入的结构化映射
- 帮助判断哪些修复项值得优先对照本地现象
- 帮助设计下一轮 before/after 验证

这张表当前**不能做**的事：
- 直接把某条 release 修复项写成本地问题根因
- 直接把某条 release 修复项写成“本地已被上游解决”
- 直接作为升级/回滚/默认行为调整的放行依据

---

## 2. 证据等级口径

- **L0｜仅同域**：问题域相似，但没有更强对应证据
- **L1｜弱对应**：除同域外，现象结构、触发位置或上下文有一定贴合
- **L2｜中对应**：除结构贴合外，已有本地样本指向同一链路/同一段状态转换
- **L3｜强对应**：已完成较明确的 before/after 或链路级证据对齐

> 当前这张表原则上不主动给 L3；除非后续有更硬的本地验证证据。

## 4. 当前最值得优先追的 4 条映射

### A. Live model switching ↔ Curie `/models` 历史
- 原因：
  - release 直接修“live switch 误触发”
  - Curie 明确存在 Discord 中 `/models` 切换历史
- 当前口径：
  - **高相关，不定因**

### B. Reply delivery / final_answer buffering ↔ 本地“假在线 / 无收口”
- 原因：
  - 本地最核心体感之一就是“任务像没收口”
  - release 直接碰 reply delivery / buffering
- 当前口径：
  - **同链路高相关，需补 delivery 层证据**

### C. Auto-reply lifecycle ownership ↔ 新旧 session 差异
- 原因：
  - Curie 已做出新旧 session 差异
  - release 明确碰 session rotation / ownership
- 当前口径：
  - **应列为下一轮重点映射项**

### D. Tool result retention fixes ↔ toolResult 后不续跑
- 原因：
  - 本地已明确有 toolResult 回写成功但 assistant 不续跑样本
  - release 直接碰 tool result retention / truncation / compaction 邻接
- 当前口径：
  - **同链路高相关，但仍未坐实是同一故障点**

---

## 5. 当前不建议硬拉映射的项

以下项当前不要硬写成“本地强相关主因”：

1. **Heartbeat busy-lane wake retry**
- 目前更像治理背景项，不是 Curie 样本的直接主线证据

2. **Cron replay 修复**
- 对 fenxue cron 链可能有价值，但对 Curie 主会话样本当前过远

3. **GPT-5 / Codex faster action + retry**
- 容易和“计划不执行”混淆；当前本地主问题是 continuation timeout，不宜混案

---

## 6. 下一步验证建议

### 第一优先：补链路更近的 before/after
1. `/models` 切换后的旧 session vs 新 session
2. toolResult 小结果 vs 大结果 / 复杂结果
3. 纯 user after vs toolResult after
4. Discord vs 非 Discord

### 第二优先：补事件层证据
1. 是否已续跑但 delivery 失败
2. 是否未续跑且直接 idle timeout
3. 是否伴随 session rotation / compaction / ownership 变化

### 第三优先：暂不做系统级结论
在上述 before/after 前：
- 不把任何一条 release 修复项写成本地根因
- 不把任何一条 release 修复项写成已自动修复依据

---

## 7. 一句话收口

> 当前 `v2026.4.5` release 中，**Live model switching / reply delivery / auto-reply lifecycle ownership / tool result retention** 四条与本地现象最值得继续做映射验证；但截至目前，它们仍然属于“高相关映射项”，不属于“已完成本地定因”的修复项。
