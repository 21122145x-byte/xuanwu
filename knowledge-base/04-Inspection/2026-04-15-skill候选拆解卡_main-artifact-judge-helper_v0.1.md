## 一、为什么现在补它

`main-artifact-judge-helper` 值得现在补，是因为“主产物优先机制”已经在主文中成立，但目前仍缺快速判定与预警层：

1. **触发条件清楚**
   - 当前阶段同时出现多个产物候选时
   - 主文、规则卡、配件、说明件并行生长时
   - 需要判断阶段结果默认该归位到哪里时

2. **输入输出相对稳定**
   - 输入：当前阶段目标、当前产物列表、每个产物的作用与归位位置
   - 输出：主产物候选判断、配件识别、归位建议、预警提示

3. **收益明显**
   - 能减少配件反客为主
   - 能降低“正文没长，配件先膨胀”的风险
   - 能帮助主入口位更稳定锁定唯一主产物

4. **边界必须提前卡死**
   - 它适合辅助判断和预警
   - 不适合自动替代主入口位做最终主产物裁决
   - 更不适合自动决定哪些配件一律删掉

所以它属于：
> **适合做主产物判断辅助 skill，但不能做自动主产物裁判**

## 二、对应的上位机制

这个 skill 候选主要承接：

### 1. 主产物优先机制
来源：
- 单一入口式 multi-agent 协作框架主文（单元 6）

当前稳定口径：
- 当前阶段应锁定唯一主产物
- 配件可以存在，但不能反客为主
- 主产物应是阶段结果默认优先归位点

### 2. 当前缺口方向
当前已明确仍缺：
- 主产物判定表
- 配件反客为主预警表
- 资产上提前置条件表

所以这张 skill 候选，本质上是把这些缺口往“辅助判断器”方向收。

## 三、这个 skill 到底做什么

### 建议 skill 名称
- `main-artifact-judge-helper`

### skill 的最小职责
只做下面三件事：

1. **检查**
   - 当前是否已明确唯一主产物
   - 当前是否存在配件膨胀 / 反客为主风险
   - 当前阶段结果默认归位点是否清楚

2. **补全**
   - 输出一版主产物候选判断草稿
   - 输出配件 / 说明件 / 辅助件的初步区分建议

3. **规范化**
   - 将散乱的“主文 / 配件 / 说明件”判断整理成统一结构块

### 明确不做的事
这个 skill 不应：
- 自动决定最终主产物是谁
- 自动否定某配件的存在价值
- 自动决定某对象是否上提为独立资产
- 替代主入口位做最终归位裁决
- 把“当前最像主产物”误写成“已正式锁定主产物”

一句话：
> 它是主产物判断辅助器，不是主产物裁判。

## 四、建议的输入 / 输出

### 输入（最小版）
至少支持：
1. 当前阶段目标
2. 当前产物列表
3. 每个产物的作用说明
4. 当前已知归位位置 / 目标用途

### 输出（最小版）
建议支持：
1. 主产物判断缺项检查结果
2. 主产物 / 配件 / 说明件初步区分草稿
3. 归位建议与预警提示

---

## 五、为什么适合做成 skill

因为它是典型的：
- 高频判断辅助动作
- 错误模式稳定
- 输出结构相对固定
- 人工复核成本可控
- 一旦做对，对主线收束质量帮助很大

如果没有它，最容易出现：
- 说明件越写越厚，主文反而没长
- 配件先变成大家默认在看的东西
- 当前阶段结果不知道该归位到哪
- 主入口位临场判断时被长配件带偏

---

## 六、为什么不能全自动做主产物裁决

即使这个 skill 候选值得做，也必须明确：

### 不能自动做的判断
- 当前阶段真正唯一主产物到底是谁
- 某配件是否只是当前轮辅助件，还是即将上提为独立资产
- 某说明件是否其实承担了主产物功能
- 当前是否需要延迟主产物裁决，先补主文骨架

所以：
- **判断结构完整** ≠ **主产物裁决成立**
- **看起来最像主产物** ≠ **已经应被正式锁定**

它最合理的定位是：
> **主产物判断器 / 配件预警器 / 归位辅助器**
而不是：
> **自动主产物裁判**

## 八、建议的触发语境

适合触发：
- “帮我判断当前主产物更像哪个”
- “检查这轮是不是配件开始反客为主了”
- “把当前这些产物整理成主产物 / 配件 / 说明件结构”
- “当前阶段结果默认应该归位到哪里”

不适合触发：
- 当前阶段目标本身还没锁定
- 当前需要的是最终内容裁决，而不是辅助判断
- 当前对象列表本身还极不清楚

---

## 九、当前建议的最小试跑方式

### 试跑样本建议
选 4 类样本：
1. 主文明显是主产物的样本
2. 配件反客为主风险样本
3. 说明件承担过多主线功能的样本
4. 归位位置不清的样本

### 试跑要看什么
- 能否稳定识别主产物判断缺项
- 能否识别配件膨胀风险
- 会不会过早把对象写死为主产物
- 会不会把说明件误判为配件或反之

### 最小通过标准
- 至少 4 个样本无明显误写“主产物已正式锁定”
- 至少 3 个样本输出明显优于手工临场判断
- 不出现“把配件误收成主产物”的重大问题
