# Anna 审计承接模板 v1

- 适用协议：`/Users/xiaojingbo/.openclaw/workspace/knowledge-base/04-Inspection/2026-04-11-半自动任务执行协议_v1.md`
- 上位规则：`/Users/xiaojingbo/.openclaw/workspace/knowledge-base/01-Constitution/跨Agent协作流转执行卡.md`
- 用途：统一记录 Anna 对半自动任务的审计承接、放行、打回、阻塞与回执动作

## 使用原则
1. `sent-to-audit` 不等于已承接；只有 Anna 显式改写状态后，才算接住。
2. 审计承接、打回、放行、阻塞，都必须落到显式状态与字段，不靠聊天语气表达。
3. 跨 Agent 审计交接完成后，默认要求以 `session_send` 形式回复【收到】或回传结果，防止断链。
4. `accepted` 只代表当前审计阶段放行，不等于已回传、不等于已 closed。

## 审计位最小动作清单
- 收到 `sent-to-audit` 后，不要只口头回复，必须落 `audit-in-progress` / `returned` / `audit-blocked` 之一。
- 若已完成跨 Agent 回执，默认补一次 `session_send` 提醒，要求下一责任位显式回复承接。
- 若同一任务在同一环节连续 3 次 `returned → 修订 → returned`，应提示触发熔断并升级粉雪。
- 若状态进入 `awaiting-human-confirmation`，不得沿用短阈值；必须显式写超时阈值与升级对象。

## 一句话口径
> Anna 的审计动作不只是“给意见”，而是把 `sent-to-audit` 明确接成 `audit-in-progress / returned / audit-blocked / accepted` 这些可执行状态，并通过 `session_send` 回执把链路接稳。
