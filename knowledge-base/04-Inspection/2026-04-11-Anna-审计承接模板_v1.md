# Anna 审计承接模板 v1

- 适用协议：`/Users/xiaojingbo/.openclaw/workspace/knowledge-base/04-Inspection/2026-04-11-半自动任务执行协议_v1.md`
- 用途：统一审计承接、阻塞、打回、放行写法

## A. 审计承接

```md
status: audit-in-progress
owner: anna
next_owner: anna
next_action: audit
updated_at: <YYYY-MM-DD HH:mm Asia/Shanghai>
artifact_where:
- <主文路径>
- <supporting artifacts>
known_issues:
- <当前先看到的问题>
decision_basis:
- 已收到最小承接包
- 审计范围明确
handoff_mode: main-agent-send
blocking_reason:
```

## B. 审计阻塞

```md
status: audit-blocked
owner: anna
next_owner: <curie / other>
next_action: <补材料 / 重送审>
updated_at: <YYYY-MM-DD HH:mm Asia/Shanghai>
artifact_where:
- <当前收到的材料>
known_issues:
- <缺什么>
decision_basis:
- 当前信息不足，无法有效审计
handoff_mode: main-agent-send
blocking_reason: <缺失材料 / 范围不清 / 输入不完整>
```

## C. 审计打回

```md
status: returned
owner: anna
next_owner: <curie / worker>
next_action: revise
updated_at: <YYYY-MM-DD HH:mm Asia/Shanghai>
artifact_where:
- <被打回文档>
known_issues:
- <核心问题>
decision_basis:
- 当前内容不满足放行条件
handoff_mode: main-agent-send
blocking_reason:
return_reason: <为什么打回>
minimum_fix_path: <最小修复路径>
who_should_rework: <谁来改>
reentry_condition: <满足什么后可重回审计>
```

## D. 审计放行

```md
status: accepted
owner: anna
next_owner: <curie / announcer>
next_action: announce | archive
updated_at: <YYYY-MM-DD HH:mm Asia/Shanghai>
artifact_where:
- <放行文档>
known_issues:
- <仍保留的限制>
decision_basis:
- 当前阶段审计通过
handoff_mode: main-agent-send
blocking_reason:
accept_scope: <本次放行覆盖范围>
accept_basis: <放行依据>
known_limits: <已知边界>
```
