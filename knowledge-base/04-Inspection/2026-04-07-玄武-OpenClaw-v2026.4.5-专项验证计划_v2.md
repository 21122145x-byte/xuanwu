## 5. 事件级拆分（Anna 要求显式并入 v2）

本节为 v2 强制项，不能省。

### 事件类型 E1｜未续跑
定义：
- user / toolResult 已进入会话
- assistant 后续未继续产出
- 或直接出现 `LLM idle timeout (60s)`

### 事件类型 E2｜已续跑未投递
定义：
- assistant 实际有后续生成迹象
- 但用户侧未看到、或消息未正常投递/回显

### 事件类型 E3｜已投递但观感失真
定义：
- 真实事件链存在
- 但 transcript / render / surface 观感让人误以为“断流/假在线”

### 每条样本必须额外记录的 3 个事件邻接项
1. 是否伴随 `session rotation`
2. 是否伴随 `reply ownership` 变化
3. 是否伴随 `active run` 边界错位迹象

在 v1 的 A/B 基础上，v2 扩成“变量 × 事件”的矩阵。

| 组别 | Surface | Session | `/models` 历史 | Trigger Type | ToolResult Complexity | 目标 |
|---|---|---|---|---|---|---|
| A | Discord | 旧 | 有 | user | 无 | 继续盯旧样本 user-after 是否失续跑 |
| A2 | Discord | 旧 | 有 | toolResult | 轻 | 继续盯 toolResult-after 是否失续跑 |
| A3 | Discord | 旧 | 有 | toolResult | 中 | 看复杂度是否显著放大 |
| B | Discord | 新 | 尽量同线 | user | 无 | 继续对照旧 vs 新 |
| B2 | Discord | 新 | 尽量同线 | toolResult | 轻 | 看新 session 是否降低复现率 |
| C | 非 Discord | 新 | 尽量同线 | user / toolResult | 轻 | 看 Discord 是否为放大环境 |
| D | Discord | 新 | 不同模型线 | user / toolResult | 轻 | 看模型线路是否显著影响复现 |

> 原则：一次执行时只改一个主变量，不把 surface / session / model line 混改。
