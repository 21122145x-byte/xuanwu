## 一、目标

当前这份清单只回答四个问题：

1. **当前真实运行版本是什么**
2. **当前真实加载路径是什么**
3. **当前真实 memory slot 绑定是什么**
4. **当前真实 config 兼容状态是什么**

一句话：
> **先钉事实，再谈升级。**

---

## 二、必须核死的 4 组事实

## 1. 当前真实版本
### 必须回答
- 当前运行中的 memos-local-openclaw-plugin，到底是 `1.0.3`、`1.0.7`，还是其他版本？

### 必须取证来源
至少同时核：
1. `plugins.installs.memos-local-openclaw-plugin.version`
2. 当前实际加载目录内的 `package.json` / version 字段
3. `openclaw status` 或 plugin 列表中的版本显示
4. gateway / plugin register 日志中是否带版本号

### 输出要求
必须区分：
- 安装记录版本
- 实际加载目录版本
- 运行时显示版本

若三者不一致，不允许直接收成一句“当前版本 = X”。

---

## 2. 当前真实加载路径
### 必须回答
- 当前 OpenClaw 真正加载 MemOS plugin 的路径，到底是哪一个？

### 容易混淆的路径类型
必须明确区分：
1. npm 包来源路径
2. 全局 node_modules 路径
3. `~/.openclaw/extensions/...` 当前加载路径
4. 历史 hold / backup 路径

### 必须取证来源
至少同时核：
- `plugins.installs.memos-local-openclaw-plugin.installPath`
- `openclaw status` / plugin source 列
- 当前实际存在的插件目录
- gateway 启动日志中的 plugin source

### 输出要求
必须明确写成：
- **发布来源**
- **安装路径**
- **当前真实加载路径**

禁止把三者混成一个“安装路径”。

---

## 3. 当前真实 slot 绑定
### 必须回答
- 当前 memory slot 是否仍绑定 memos-local-openclaw-plugin？

### 必须取证来源
至少核：
- `plugins.slots.memory`
- `plugins.allow`
- `plugins.entries["memos-local-openclaw-plugin"].enabled`
- 若存在其他 memory plugin，当前是否有竞争/替代关系

### 输出要求
必须明确写出：
- 当前 memory slot 绑定对象
- 当前 plugin 是否处于 enabled
- 当前是否仍为默认 memory provider

---

## 4. 当前 config 兼容状态
### 必须回答
- 当前 `openclaw.json` 中 MemOS 配置，是否与拟升级目标版本兼容？

### 必须核的关键项
至少核：
1. embedding provider 配置
2. summarizer provider 配置
3. auto-recall / allowPromptInjection 相关配置位
4. memorySearch / context engine 相关兼容项
5. 升级后是否仍需要显式关闭或保持某些旧配置

### 输出要求
把结果分成三类：
- **兼容，可直接沿用**
- **兼容但建议收紧**
- **不兼容，升级前必须改**

禁止只写“看起来差不多能用”。

---

## 三、功能状态表述要求

在事实核验版里，禁止直接写：
- “核心功能正常工作”
- “当前运行正常”
- “升级风险低”

除非已经明确区分以下层次：
1. **能触发**
2. **能运行**
3. **边界正确**
4. **长期健康**

推荐表述：
> **基础召回 / 注入链存在且可触发，但 recall 健康性、边界控制与污染风险仍有既有问题，不得写成整体正常。**

---

## 四、核验完成后的允许输出

只有当以上 4 组事实核死后，才允许进入下一步文档：

### 可进入
- `MemOS 升级建议（事实核验后版本）`
- `MemOS 升级执行卡`
- `MemOS 升级后验证清单`

### 当前仍不可进入
- 直接批准升级
- 直接执行升级
- 直接写“升级风险低、可上”

---

## 五、最小交付格式

建议最终交付至少包含：

### 1. 事实核验结论表
- 当前真实版本
- 当前真实加载路径
- 当前真实 slot 绑定
- 当前 config 兼容状态

### 2. 证据来源表
- 配置来源
- 日志来源
- 目录来源
- 运行时来源

### 3. 升级前判断
- 当前是否具备进入升级建议版的条件

---

## 六、一句话收口

> **这份清单的作用不是继续讨论版本差异，而是先把“当前真实在跑什么、从哪里跑、怎么绑着跑、配置是否兼容”四件事钉死；在这之前，不进入升级执行判断。**
