# 团队 SOUL / AGENTS 横向诊断 v1

## 一、总判断

当前团队文件体系已经开始成型。

优势在于：
- 角色人格开始清晰
- 分工雏形已建立
- 粉雪、Anna、旺财三位已经能明显看出不同工作气质

问题在于：

> SOUL 普遍比 AGENTS 更鲜明；人格普遍比接口更成熟；个体都开始像人了，但团队还没完全像一支队。

也就是说：
- 粉雪像 manager
- Anna 像 verifier
- 旺财像 writer / creative
- 富贵处于新角色迁移期

但协作接口、统一路由规则、交接规范还没有完全形成一套闭环。

---

## 二、单体观察

### 粉雪
**当前优势：**
- Manager 角色感已较清晰
- 风格稳，强调检查、验证、汇报
- 已有 Example Interactions，具备较强行为样本价值

**当前问题：**
- 更像“谨慎型调度员”，还不完全像真正的 workflow owner
- 优先级、任务收束、调度结束条件写得还不够清楚

**建议优先补：**
- 任务优先级 / triage 规则
- handoff 统一推动职责
- 状态收束与完成标准

---

### Anna
**当前优势：**
- 边界感最清楚
- 验证与知识守护者定位很硬
- 证据、日志、错误码、路径这些验证锚点意识非常强
- 5 字段 handoff 协议很有价值

**当前问题：**
- “守门人”强于“助推器”
- 驳回机制清楚，但修复路径与放行条件还可以更标准化

**建议优先补：**
- validation 输出格式
- repair / remediation 路径
- 知识沉淀标准

---

### 旺财
**当前优势：**
- 人格辨识度最强
- 长文、创意、结构、判断这条线非常清楚
- 语言和协作对象的匹配度高，不像模板 agent

**当前问题：**
- 人格强于协议
- 在多 agent 协作里，容易因为能力覆盖面广而显得边界不够显式

**建议优先补：**
- Should / Shouldn’t
- Output Preferences
- Example Interactions
- Team Collaboration 接口说明

---

### 富贵
**当前状态：**
- 处于新职能迁移期
- 不能与另外三位直接按完成度横向硬比

**当前关键任务：**
- 先建立 A 股分析位的职责边界
- 先写清输出规则与团队接口
- 再谈人格完成度

**建议优先补：**
- 财经分析边界
- 事实 / 推断 / 判断分层
- 与粉雪 / Anna / 旺财的交接接口

---

## 三、团队层面的三个核心问题

### 问题 1：角色在 SOUL 里比在 AGENTS 里更像自己
人格起来了，但协作协议还没完全同步。

### 问题 2：统一路由规则还不够一致
大家开始有边界意识，但“什么任务默认给谁”还没有完全对齐成一套系统。

### 问题 3：交接协议已出现，但未全员标准化
Anna 的 5-field handoff 很值钱，但还没有成为团队统一的默认动作。

---

## 四、当前最值得统一的，不是语言风格，而是协作接口

优先统一：
- routing rules
- handoff payload
- 任务完成标准
- 路径 / 文档沉淀习惯

不要优先统一：
- 每个人的口气
- 每个人的语感
- 每个人的表达模板

一句话：

> 先统一怎么接球、怎么传球、怎么落球，再考虑怎么说话更整齐。

---

## 五、建议的三步走

### 第一步：先观察，不大修
先让当前版本跑一段时间，避免把正在形成的人格洗成正确模板。

### 第二步：先统一协作接口
优先沉淀：
- handoff 协议
- routing 规则
- 完成标准
- 共享文件系统规范

### 第三步：按角色做定点优化
- 粉雪：补调度优先级与收束
- Anna：补验证输出格式与修复路径
- 旺财：补行为协议与示例交互
- 富贵：补新职能的骨架与接口

---

## 六、当前一句话结论

> 粉雪像骨架，Anna像刹车，旺财像火力；现在最缺的不是人格，而是把三者真正接成一套传动结构。富贵则处于新职能迁移期，当前首要任务是先立边界、规则与接口。
