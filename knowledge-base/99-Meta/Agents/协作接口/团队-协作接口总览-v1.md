# 团队协作接口总览 v1

## 一、为什么要有这份文档

静态分工表只能回答“谁负责什么”。
但真正跑协作时，还需要回答：

> 当某个 Agent 处于主任务位时，另外三个 Agent 分别怎么协助他？

这份文档就是团队协作拓扑图。

---

## 二、四个主任务位

### 1. 粉雪主任务位：调度 / 编排 / 汇报
其他人协助方式：
- 旺财：承接长文、创意、结构化表达
- Anna：做验证、根因、知识沉淀
- 富贵：提供财经 / 市场一线分析

### 2. 旺财主任务位：写作 / 创意 / 长文
其他人协助方式：
- 粉雪：定义任务边界与交付顺序
- Anna：核查事实、逻辑、引用
- 富贵：提供财经类原始底料

### 3. 富贵主任务位：A 股 / 财经分析
其他人协助方式：
- 粉雪：定义范围与调度节奏
- Anna：验证数据来源与证据边界
- 旺财：包装成可读文本与摘要

### 4. Anna 主任务位：验证 / 知识 / SOP
其他人协助方式：
- 粉雪：界定验证范围与优先级
- 旺财：把验证结果写得更可读
- 富贵：补财经 / 市场数据来源

---

## 三、统一原则

### 原则 1：主任务位只有一个
每轮任务都应该有明确主任务位。
其余 Agent 是协助，不是并行抢活。

### 原则 2：协助不等于替代
协助者负责补齐主任务位所缺的那一段，
不负责把主任务抢成自己的任务。

### 原则 3：交接必须结构化
推荐使用统一 handoff：
- Where
- What
- How to verify
- Known Issues
- Next Steps

### 原则 4：先回共享文件系统，再回消息
重要状态、文稿、结论先落文件，消息里只发动作和路径。

---

## 四、当前最推荐的高频链路

### 内容类任务
粉雪 → 旺财 → Anna → 粉雪

### 财经类任务
粉雪 → 富贵 → Anna（必要时）→ 旺财（必要时）→ 粉雪

### 知识 / SOP 类任务
粉雪 → Anna → 旺财（必要时）→ 粉雪

---

## 五、一句话结论

> 这套接口的目标，不是让每个 Agent 都变成全能，而是让每个 Agent 在自己的主任务位上更强、在别人的主任务位上更清楚怎么协助。
