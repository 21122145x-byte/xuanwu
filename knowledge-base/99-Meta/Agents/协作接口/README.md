# Agent 协作接口与优化文档导航

## 一、这套文档是干什么的

这组文档不是静态分工表，而是团队协作的运行骨架。
它解决三层问题：

1. **谁负责什么**（主任务位）
2. **别人怎么协助他完成**（协作接口）
3. **后续该怎么优化**（下一步优化建议）

一句话：

> 让团队从“有角色”推进到“能协作”。

---

## 二、总览顺序（推荐阅读）

### 第一步：先看团队总文档
- `团队-协作接口总览.md`
- `团队-标准协作协议.md`

这两份解决：
- 协作拓扑图
- 标准动作
- 统一原则

---

### 第二步：再看标准模板
- `团队-任务发起模板.md`
- `团队-Handoff-模板.md`
- `团队-完成标准模板.md`

这三份解决：
- 任务怎么发
- 交接怎么写
- 什么叫真正完成

---

### 第三步：看每个 Agent 的协作接口卡
- `粉雪-协作接口卡.md`
- `旺财-协作接口卡.md`
- `富贵-协作接口卡.md`
- `Anna-协作接口卡.md`

这四份解决：
- 当某个 Agent 是主任务位时，其他人怎么协助他
- 什么不能替他做
- 产出后谁来接

---

### 第四步：看每个 Agent 的下一步优化建议
- `粉雪-SOUL-AGENTS-下一步优化建议-v1.md`
- `旺财-SOUL-AGENTS-下一步优化建议-v1.md`
- `富贵-SOUL-AGENTS-下一步优化建议-v1.md`
- `Anna-SOUL-AGENTS-下一步优化建议-v1.md`

这四份解决：
- 每个 Agent 现在最强的地方
- 最大短板在哪里
- 后续应该先补什么

## 四、当前建议用法

## 五、一句话结论

> 这套文档不是为了把每个 Agent 写得更像模板，而是为了让每个 Agent 在自己的主任务位上更强、在别人的主任务位上更清楚怎么协助。
