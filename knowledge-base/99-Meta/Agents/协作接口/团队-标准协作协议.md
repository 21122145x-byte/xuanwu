# 团队标准协作协议 v1

## 一、目标

这份协议用于统一多 Agent 协作时的基础动作，解决以下问题：
- 任务发起时信息不全
- 交接时只说“你看一下”
- 协助与抢活边界不清
- 做完后没人知道怎么算完成

一句话：

> 先统一怎么接球、怎么传球、怎么落球。

---

## 二、任务发起模板

发起任务时，至少要包含：

- **任务目标**：这件事最终要产出什么
- **主任务位**：这轮谁是主责 Agent
- **协助位**：谁负责补哪一段
- **交付形式**：文档 / 摘要 / 报告 / 校验结果 / 口径
- **时间要求**：是否紧急、是否分阶段交付
- **约束条件**：不能碰什么、必须保留什么、要遵循什么格式

---

## 三、标准 handoff 模板

推荐统一使用 5-field handoff：

- **Where**：产出路径 / 文件位置
- **What**：做了什么核心动作
- **How to verify**：接收方应重点核什么
- **Known Issues**：已知问题 / 不确定项
- **Next Steps**：下一步希望谁做什么

禁止只发：
- “做完了”
- “你看下”
- “差不多了”

---

## 四、协助规则

### 原则 1：每轮只有一个主任务位
其余人是协助，不是并行抢活。

### 原则 2：协助不等于替代
协助者只补主任务位缺的那一段，不接管整轮任务。

### 原则 3：先落文件，再发消息
重要状态、结果、文稿、结论，优先沉淀到共享文件系统，消息里只发动作和路径。

### 原则 4：边界优先于热情
能帮，不代表该抢。先问这是不是自己那一段职责。

---

## 五、完成标准模板

一轮任务完成，至少要满足：

- 结果已落文件
- 主任务位交代清楚产出内容
- 协助位已完成自己负责部分
- 已知问题被显式暴露
- 下一步归属明确

如果以上任一项缺失，不算真正完成。

---

## 六、高频协作链路

### 内容类任务
粉雪发起 → 旺财主写 → Anna核验 → 粉雪收束

### 财经类任务
粉雪发起 → 富贵分析 → Anna校验（必要时）→ 旺财包装（必要时）→ 粉雪收束

### 知识 / SOP 类任务
粉雪发起 → Anna主责 → 旺财优化表达（必要时）→ 粉雪收束

---

## 七、一句话结论

> 好协作不是每个人都很能干，而是每个人都知道什么时候该接、接哪一段、做到什么程度、再交回给谁。
