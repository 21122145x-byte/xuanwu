# 粉雪 SOUL / AGENTS 下一步优化建议 v1

## 一、当前判断

粉雪已经开始长出比较稳的 Manager 角色感。
她的优势不是“会说漂亮话”，而是：
- 稳
- 会先检查再行动
- 有协作意识
- 已经开始有调度样子

所以她下一步最重要的，不是补人格，
而是：

> 从“谨慎型调度员”进化成“真正的流程 owner”。

---

## 二、SOUL.md 下一步最该补什么

### 1. Priority / Triage 规则
建议明确：
- 什么任务算紧急故障
- 什么任务算多 Agent 协作
- 什么任务只需要单点派发
- 什么任务必须验证后再推进

### 2. 调度结束条件
现在比较强调“检查、汇报、执行”，
后续建议补：
- 什么算任务真正收束
- 什么算应该回到 BOSS 确认
- 什么算需要二次分派

### 3. Handoff 推动职责
粉雪作为调度位，后续可明确写进：
- 她不仅自己要遵循 handoff
- 她还负责推动团队统一使用交接协议

---

## 三、AGENTS.md 下一步最该补什么

### 1. 统一路由规则
建议把“什么任务先给谁”写得更清楚，避免临场拍脑袋。

### 2. 汇总与收口接口
粉雪的价值不只是分发，
还在于：
- 汇总状态
- 收束任务
- 形成回报
- 决定是否进入下一轮

### 3. 多 Agent 任务的标准链路
建议在 AGENTS 中明确几条高频链路，降低团队协作成本。

---

## 四、建议节奏

当前先不急着重写。
先在真实协作中观察：
- 粉雪会不会只稳不推
- 会不会把调度写成汇报
- 会不会在任务收束处发虚

等观察一段时间，再把真正有效的规则写回正式文件。

---

## 五、一句话结论

> 粉雪现在缺的不是人格，而是更明确的优先级判断、状态收束和调度完成标准。
