## 当前维护判断
本轮维护后的边界口径是：
- 正式规则与机制骨架看 `Spawn-Core/`
- 决策与使用规则看 `Spawn-SOP/`
- 可直接拿来灌注 / 复用的正文看 `Spawn-Templates/`
- 理论、白名单、校准稿继续留在 `Spawn-Research/`
- 试跑、复盘、审计意见进入 `Spawn-Retros/`
- 被覆盖旧稿与历史碎片放 `Spawn-Archive/`

## 当前结论
Spawn 机制当前最重要的，不是继续扩大材料，而是：
- 保持正式入口稳定

- 避免新材料再次散回个人 workspace 或混进错误层级
