# Spawn机制 V0｜启动条件

- 版本：V0
- 状态：骨架版 / 待首轮试跑校正
- 维护者：Anna
- 参与建议：粉雪（调度）、旺财（结构与表达）、相关执行 Agent
- 目的：把“什么时候应该启动 Spawn”从经验判断压成可执行规则，作为团队第一版派单入口。

---

## 一、文档目的

本文件只回答一件事：

> **一个任务在什么情况下，应该从“单 Agent 直接做”切换为“进入 Spawn 机制处理”。**

它不是模型能力说明书，也不是最终调度器实现文档。
它的作用是给主控位一个**可落地的启动判断**，避免继续靠拍脑袋决定是否 Spawn。

---

## 二、V0 设计原则

### 原则 1
**Spawn 不是为了把任务拆得更碎，而是为了切断单 Agent 长链后程失真。**

### 原则 2
**只要任务结构已经具备“后段容易掉质”的条件，就优先考虑 Spawn。**

### 原则 3
**V0 采用保守阈值。宁可多触发几次，也不要继续把明显高风险任务硬塞给单 Agent。**

---

## 三、触发条件（建议进入 Spawn）

出现以下任一情况时，建议主控位评估是否进入 Spawn：

- 阶段数 **> 2**
- 同时包含：抽样 / 分类 / 统计 / 归因 / 汇总 中的多类动作
- 需要跨阶段对账
- 最终结论需要可审计、可回溯
- 错判会触发返工、错误决策或错误暂停
- 任务已知存在“前段可做、后段汇总容易漂”的风险
- 执行模型或执行者已被观察到存在后程稳定性边界
- 任务输出不仅是局部事实，还要形成最终放行 / 返工 / 暂停判断

---

## 四、默认必须 Spawn 的判定规则

### V0 保守判定
若一个任务同时满足以下条件中的 **3 条及以上**：

- 阶段数 > 2
- 既要抽样统计，又要归因判断
- 需要跨阶段对账
- 最终结论需要可审计
- 错判会造成返工或误决策
- 已知模型存在后程稳定性边界

则：

> **默认必须进入 Spawn 机制**

除非主控位明确记录豁免理由。

---

## 五、可量化保守阈值（基于当前样本）

> 注：以下阈值是当前基于真实事件得出的保守运营阈值，不代表理论极限；后续应根据试跑样本继续校正。

### 1. 单次 query 分类
- 建议上限：**≤ 15 条**

### 2. 单次 Top N 结果组审计
- 建议上限：**≤ 6 组**

### 3. 单次边界样本审计
- 建议上限：**≤ 6 组**

超过以上任一上限时，不建议继续按“小单任务”处理，应由主控重新评估是否拆入 Spawn。

---

## 六、适合优先进入 Spawn 的任务类型

当前 V0 优先推荐以下任务带：

- 审计类
- 验证类
- RCA / 排查类
- 结构化返工类
- 需要多阶段口径统一的治理类任务
- 会形成正式结论或操作建议的评估类任务

这些任务的共同特点是：
- 局部事实可以拆
- 最终判断不能乱下放
- 后段整合层最容易失真

---

## 七、暂不优先进入 Spawn 的任务类型

以下类型暂不作为 V0 首批重点适用对象：

- 高创意发散类任务
- 问题边界极不清晰的开放题
- 强依赖连续直觉流动的长篇创作
- 高度个人化、很难标准化验收的纯灵感型任务

不是这些任务永远不能 Spawn，
而是 **V0 阶段先不要用它们做规则样本。**

---

## 八、豁免条件

即使触发了建议条件，在以下情况下，主控位可暂时豁免 Spawn：

- 任务总量虽大，但本轮只要求一个极小局部闭环
- 输出仅供内部探索，不形成正式决策
- 返工成本极低，且允许试错
- 已由高稳定主控实时分段看护，风险被显著压低
- 当前时间窗口极短，拆分成本会高于直接完成成本

### 豁免要求
凡做豁免，建议主控至少记录：
- 为什么不 Spawn
- 本轮风险是什么
- 一旦失真，谁负责收口

---

## 九、主控位启动判断流程（V0）

建议主控位在接单后按以下顺序判断：

1. 这个任务是不是单动作任务？
2. 是否超过单 Agent 稳定闭环长度？
3. 是否包含跨阶段对账或最终可审计结论？
4. 是否存在已知后程失真风险？
5. 若继续单 Agent 执行，返工成本是否可接受？

若第 2-4 项中出现多个“是”，优先进入 Spawn 评估。

---

## 十、记录格式建议

当主控判断“启动 Spawn”时，建议留下简短记录：

```md
## Spawn 启动记录
- 任务名：
- 启动原因：
- 命中条件：
- 是否命中强制 Spawn 阈值：是 / 否
- 本轮主控：
- 豁免说明（如无则留空）：
```

---

## 十一、当前已知限制

- 当前规则主要基于少量真实样本，不是大样本统计
- 量化阈值仍偏保守，后续需要校正
- 不同模型 / 不同 Agent 的稳定边界可能不同
- 目前更多是在验证“结构风险”，不是在测模型极限
