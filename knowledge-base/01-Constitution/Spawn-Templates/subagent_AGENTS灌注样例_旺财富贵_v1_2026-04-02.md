## 一、使用原则

spawn 前先判断三件事：
1. 这是执行型，还是评估型？
2. 这次是否需要正式验收链？
3. 任务边界是否足够清楚，还是必须叠加子任务卡？

### 快速选择
- **执行型 + 正式交付** → 执行者版 + 子任务卡
- **评估型 + 正式验收** → 评估者版 + 子任务卡
- **执行型 + 低风险短任务** → 轻量执行者约束
- **评估型 + 快速复核** → 轻量评估者约束

## 二、旺财可复制样例

### 场景 A：长文 / 方案某一段重写
**推荐组合：执行者版 + 子任务卡**

#### 注入给 subagent 的 AGENTS 模板
使用：
- `subagent_AGENTS_执行者版.md`

#### 搭配的子任务卡应明确
- 这次只重写哪一段
- 不准扩写哪些部分
- 输出是段落、提纲还是完整页
- 回传格式
- 是否要进入 Anna / BOSS 验收链

#### 适用原因
这种任务属于：
- 要产出正文
- 边界容易漂
- 输出需要正式回传

所以不能只灌 AGENTS，不叠加任务卡。

---

### 场景 B：对现有方案做站得住性判断
**推荐组合：评估者版 + 子任务卡**

#### 注入给 subagent 的 AGENTS 模板
使用：
- `subagent_AGENTS_评估者版.md`

#### 搭配的子任务卡应明确
- 评估对象是哪份文档
- 评估维度是什么（结构 / 证据 / 风险 / 可执行性）
- 输出是放行 / 打回 / 待补证据，还是只给审阅意见

#### 适用原因
这种任务属于：
- 目标是判断成立性
- 不该替执行者重写内容
- 需要明确最小修复路径

## 三、富贵可复制样例

### 场景 A：市场材料归纳 / 前处理
**推荐组合：执行者版 + 轻量任务说明**

#### 注入给 subagent 的 AGENTS 模板
使用：
- `subagent_AGENTS_执行者版.md`

#### 如果任务边界很短
可以只加一个轻量任务说明：
- 这次只归纳哪几份材料
- 输出是简表 / 摘要 / 分类结果
- 不要求正式验收链

#### 适用原因
这种任务通常：
- 输入明确
- 输出短
- 可快速复核

如果不进入正式决策链，可以不必上完整版子任务卡。

---

### 场景 B：项目方案中的正式段落重写
**推荐组合：执行者版 + 子任务卡**

#### 注入给 subagent 的 AGENTS 模板
使用：
- `subagent_AGENTS_执行者版.md`

#### 搭配的子任务卡应明确
- 本次只改哪一节
- 原文在哪里
- 目标风格 / 目标结构是什么
- 哪些数字 / 结论不能擅自改
- 回传格式和路径

#### 适用原因
这种任务：
- 属于正式正文生产
- 容易越界扩写
- 失败成本高

因此必须叠加子任务卡。

---

### 场景 C：判断一份材料 / 一个判断是否成立
**推荐组合：评估者版 + 轻量或正式子任务卡（视风险）**

#### 注入给 subagent 的 AGENTS 模板
使用：
- `subagent_AGENTS_评估者版.md`

#### 判断规则
- 若只是快速复核 → 轻量任务说明即可
- 若结论会进入正式交付 / 审计链 → 必须叠加正式子任务卡

---

## 四、最小复制用法

### 1. 复制执行者版
适用于：
- 重写
- 归纳
- 前处理
- 资料整理
- 文件生产

### 2. 复制评估者版
适用于：
- 审计
- 验收
- 打回
- 校验
- 成立性判断

### 3. 是否叠加子任务卡
看三点：
- 是否正式交付
- 是否边界复杂
- 是否失败成本高

只要任一为“是”，优先叠加子任务卡。

---

## 五、给主 agent 的一句提醒

> **先判“做”还是“判”，再判这次是轻量任务还是正式交付。不要把 AGENTS 模板当任务卡，也不要让 subagent 只拿任务卡不拿角色约束。**
