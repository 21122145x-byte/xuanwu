# Spawn 子任务卡模板 V0

- 版本：V0
- 状态：骨架版 / 可直接试填
- 维护者：Anna
- 目的：统一 Spawn 机制下子任务的下发格式，降低临场发挥和边界漂移，保证每个子任务都能被回收、验收、复用。

---

## 一、使用说明

本模板用于：
- Spawn 主控位向子任务执行者派发任务
- 结构化定义子任务边界
- 统一子任务的输入、输出和验收标准

本模板不负责：
- 代替主控做最终判断
- 代替主控统一跨任务口径
- 代替最终总报告

一句话：

> **子任务卡的目标，是让一个子任务成为可执行、可回收、可验收的局部闭环。**

---

## 二、V0 强制字段

每张子任务卡，V0 阶段建议至少包含以下 6 个字段：

1. **任务名**
2. **目标**
3. **输入材料**
4. **输出物**
5. **边界 / 禁做项**
6. **验收标准**

如果这 6 个字段里有任一项写不清，说明这个子任务大概率还不适合派发。

---

## 三、标准模板

```md
# Spawn 子任务卡

## 1. 任务名

## 2. 目标
- 本任务要解决什么问题：
- 本任务完成后，主控应该获得什么：

## 3. 输入材料
- 文件 / 文档：
- 上下文说明：
- 已知前提：

## 4. 输出物
- 输出文件名：
- 输出形式：
- 输出内容应包含：

## 5. 边界 / 禁做项
- 本任务不处理：
- 本任务不能越界判断：
- 本任务不能修改：

## 6. 验收标准
- 完成后必须明确回答：
- 必须给出的证据 / 数字 / 结论：
- 不合格情形：
```

## 四、字段解释

### 1. 任务名
要求：
- 一眼能看出任务类型
- 一眼能看出处理对象
- 尽量避免空泛标题

推荐格式：
- `统计对账｜D/E 冲突修正`
- `底层核验｜memos 数据路径确认`
- `口径返工｜样本量状态写实修订`

---

### 2. 目标
要求：
- 写清“这次到底要解决什么”
- 不要把目标写成动作堆叠
- 最好能体现主控真正想拿到的局部结果

不推荐写法：
- “请综合分析并给出建议”

推荐写法：
- “统一 D/E 阶段统计数字，确认冲突来自填写错误还是样本归属错误。”

---

### 3. 输入材料
要求：
- 明确指定允许使用的文件或材料
- 若有关键上下文，必须写出
- 不要让子任务自行猜上下文边界

建议包含：
- 文件路径
- 阶段说明
- 已知限制
- 不必重复探索的前提

---

### 4. 输出物
要求：
- 最好指定输出文件名
- 写清输出形式
- 说明输出里至少要包含什么内容

示例：
- 输出文件名：`统计冲突核对单.md`
- 输出应包含：统一后的 D0-D3 数字、冲突来源说明、仍待确认项

---

### 5. 边界 / 禁做项
这是 V0 最关键字段之一。

要求：
- 明确写出“不要碰什么”
- 明确写出“不能替主控下什么判断”
- 明确写出“哪些文件 / 结论不要擅自修改”

如果这一项不写，子任务很容易顺手越界。

---

### 6. 验收标准
要求：
- 写成主控位可判断的标准
- 尽量避免“写得不错就算通过”
- 明确哪些结果缺失会判定不合格

推荐写法：
- “必须明确给出统一后的 D0/D1/D2/D3 数字”
- “必须说明冲突是填写错误还是样本归属错误”
- “不得越界给最终放行建议”

## 五、示例子任务卡（可直接复用）

```md
# Spawn 子任务卡

## 1. 任务名
统计对账｜D/E 冲突修正

## 2. 目标
- 统一 D/E 阶段统计数字
- 判断冲突来自填写错误还是样本归属错误

## 3. 输入材料
- D_samples.csv
- D_阶段完成报告.md
- E_最终汇报.md
- 已知问题：D1 数字在不同文件中出现冲突

## 4. 输出物
- 输出文件名：统计冲突核对单.md
- 输出应包含：
  - 统一后的 D0/D1/D2/D3 数字
  - 冲突来源说明
  - 仍待确认项（如有）

## 5. 边界 / 禁做项
- 不处理最终放行 / 返工 / 暂停结论
- 不做最终根因排序
- 不修改其他阶段结论等级

## 6. 验收标准
- 必须明确给出统一后的 D0/D1/D2/D3 数字
- 必须说明冲突是填写错误还是样本归属错误
- 不得越界给最终决策建议
```

## 六、常见错误示例

### 错误 1：目标太大
- “请完整分析这个问题并给出最终方案”

问题：这不是子任务卡，这是把主任务原封不动塞下去。

### 错误 2：输入材料不清
- “你结合现有上下文看一下”

问题：这会导致边界漂移，子任务开始自己乱补上下文。

### 错误 3：输出物不明确
- “你看着整理一下结论”

问题：无法回收，也无法验收。

### 错误 4：没写禁做项
- 子任务会自然滑向更大的解释和判断

问题：最容易污染主控层。

### 错误 5：验收标准写成感觉判断
- “写得清楚即可”

问题：不可操作。

## 七、主控位派发前自检

在发出子任务卡前，主控位建议检查：

- 这是不是一个局部闭环？
- 目标是否足够小，且足够清楚？
- 输入材料是否够用？
- 输出物是否可回收？
- 禁做项是否足够明确？
- 验收标准是否能客观判断？

若有任一项模糊，先改卡，再派发。
