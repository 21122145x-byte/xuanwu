# AI 协同条漫生产工艺白皮书（草案）v0.1

## 一、文档目的

本白皮书用于把“AI 协同条漫生产”从概念描述，压成可执行的动作链路。

它不讨论项目价值，也不讨论市场规模，
只回答一个问题：

**在 AI 协同条件下，一张条漫到底是怎么被做出来的。**

目标是让后续样本制作、流程分工、工具选择、问题复盘都能有共同参照。

---

## 二、总原则

AI 协同条漫，不是“先出图，再拼故事”，
也不是“把传统漫画流程原封不动搬进 AI”。

它的核心原则是：

1. **先定叙事，再定画面**
2. **先定阅读结构，再定美术细节**
3. **图和字分工，不重复解释**
4. **生成只是中段，筛选与修正同样重要**
5. **先做成立性验证，再追求工业化效率**

一句话说：
**AI 不是替代创作，而是把条漫生产从“重投入后验证”改成“先验证再放大”。**

## 三、标准生产链路（动作级）

### 3.1 内容切口确定

**输入：**
- 项目目标
- 当前样本任务
- 原始故事 / 设定素材

**动作：**
1. 明确本张条漫只解决一个核心表达目标
2. 写出“读者读完后最该记住什么”
3. 写出“这一张不准进入的内容”
4. 明确结尾停在什么问题、情绪或钩子上

**输出：**
- 切口定义句
- 一句话目标
- 禁入清单
- 结尾钩子说明

**验证点：**
- 是否能用一句话说清这一张在讲什么
- 是否明确知道哪些信息不该进入
- 是否存在可被感知的收束点

---

### 3.2 阅读结构拆解

**输入：**
- 切口定义句
- 原始文本素材

**动作：**
1. 把整张条漫拆成若干阅读段
2. 标记开头钩子段
3. 标记信息推进段
4. 标记停顿段 / 留白段
5. 标记尾部留钩段
6. 判断每段的信息密度是否过重

**输出：**
- 分段结构表
- 滑动节奏草图
- 段落目的说明

**验证点：**
- 是否存在明确的开头、中段、尾部结构
- 是否能看出哪里快、哪里慢、哪里该停
- 是否出现一段塞入过多信息的问题

---

### 3.3 图文分工脚本

**输入：**
- 分段结构表
- 目标信息清单

**动作：**
1. 逐段决定该段主要由图承担还是由字承担
2. 明确哪些信息必须视觉化
3. 明确哪些信息必须文字压缩
4. 明确哪些信息需要图文共同完成
5. 删除“图说一遍、字再说一遍”的重复内容

**输出：**
- 逐段图文分工稿
- 删除项清单

**验证点：**
- 是否每段都能明确“谁是主承载”
- 是否避免重复解释
- 是否存在文字负担压死画面的情况

---

### 3.4 视觉系统预设

**输入：**
- 图文分工稿
- 项目视觉方向

**动作：**
1. 提取角色视觉锚点
2. 提取场景气质关键词
3. 提取色调、材质、光感方向
4. 确定核心概念的视觉化方式
5. 确定版面语言：长短画幅、留白、密度、节奏感
6. 整理参考图方向

**输出：**
- 视觉关键词表
- 视觉锚点清单
- 参考图板 / moodboard

**验证点：**
- 是否能看出视觉统一方向
- 是否有可重复调用的锚点
- 是否避免“每张都像不同作品”

---

### 3.5 提示词与控制准备

**输入：**
- 视觉关键词表
- 分段结构表
- 图文分工稿

**动作：**
1. 为每一段设定主 prompt
2. 为整体设定统一风格 prompt
3. 写出 negative prompt
4. 统一角色描述模板
5. 决定是否启用 seed、参考图、LoRA、ControlNet
6. 明确每一段的生成目标与限制条件

**输出：**
- Prompt 包
- 角色模板
- 参数表
- 控制方案说明

**验证点：**
- 是否提示词已统一口径
- 是否每段生成目标清楚
- 是否已经提前考虑一致性控制

---

### 3.6 分段生图与首轮筛选

**输入：**
- Prompt 包
- 参数表

**动作：**
1. 按段落逐段生图，不一次性乱生
2. 每段生成多个候选
3. 按信息准确性筛一轮
4. 按气质一致性筛二轮
5. 按可连续阅读性筛三轮
6. 淘汰明显偏题或风格漂移的候选

**输出：**
- 每段候选图组
- 每段入选图
- 淘汰理由记录

**验证点：**
- 是否至少每段保留一个可进入后续修正的版本
- 是否已有初步连续感
- 是否发现风格漂移规律

---

### 3.7 风格统一与局部修正

**输入：**
- 入选图组

**动作：**
1. 修角色面部 / 服饰 / 道具一致性
2. 修场景中的重复元素与世界规则
3. 修色调统一
4. 修镜头切换中的跳感
5. 必要时局部重绘
6. 必要时回炉重生某一段

**输出：**
- 定稿图组 v1
- 修正说明单

**验证点：**
- 是否像同一部作品
- 是否人物稳定可识别
- 是否存在明显返工黑洞

---

### 3.8 文字嵌入与排版

**输入：**
- 定稿图组 v1
- 图文分工稿

**动作：**
1. 嵌入文字
2. 调整字号、信息密度、留白
3. 拼接长图
4. 检查滑动阅读感
5. 调整局部段落的节奏过密或过松问题

**输出：**
- 标准样本 v1

**验证点：**
- 是否顺滑读下去
- 是否存在局部过挤或过空
- 是否文字压住了画面

---

### 3.9 试读、记录与返修

**输入：**
- 标准样本 v1

**动作：**
1. 内部试读
2. 标记卡点、误读点、记忆点
3. 记录最强反馈与最大问题
4. 判断是否需要回到结构层、图文层或视觉层返修
5. 输出修订版

**输出：**
- 试读问题单
- 修订建议
- 标准样本 v2

**验证点：**
- 是否有人能完整读完
- 是否能复述大意
- 是否记住一个画面 / 词语 / 气质
- 是否愿意继续看下一张

## 四、传统流程 VS AI 协同流程

### 传统条漫制作流程

```text
选题 / 剧情确定
  ↓
脚本撰写
  ↓
分镜设计
  ↓
角色 / 场景设定
  ↓
草图绘制
  ↓
线稿 / 上色 / 细化
  ↓
文字嵌入与排版
  ↓
成品检查
  ↓
发布
```

### AI 协同条漫制作流程

```text
内容切口确定
  ↓
阅读结构拆解
  ↓
图文分工脚本
  ↓
视觉系统预设
  ↓
提示词与控制准备
  ↓
分段生图与首轮筛选
  ↓
风格统一与局部修正
  ↓
文字嵌入与排版
  ↓
试读、记录与返修
  ↓
发布
```

### 核心差异

- 传统流程重在绘制工序推进
- AI 协同流程重在前置结构设计与中段筛选修正
- 传统流程的风险是成本高、周期长
- AI 协同流程的风险是风格漂移、信息失衡、返工失控

## 五、工具映射建议

| 环节 | 主要动作 | 可用工具 |
|---|---|---|
| 内容切口确定 | 目标、禁入项、钩子定义 | 文档 / 白板 / 人工判断 |
| 阅读结构拆解 | 分段、节奏、留白 | 文档 / 表格 / 白板 |
| 图文分工脚本 | 逐段信息分工 | 文档 / 表格 |
| 视觉系统预设 | 风格、锚点、参考图 | Pinterest / 飞书 / 本地参考图板 |
| 提示词与控制准备 | prompt、角色模板、参数 | Midjourney / SD / ComfyUI / 文档 |
| 分段生图与首轮筛选 | 候选生成、筛图 | Midjourney / SD / ComfyUI |
| 风格统一与局部修正 | 一致性、重绘、返工 | LoRA / ControlNet / 局部重绘 / PS |
| 文字嵌入与排版 | 拼图、长图、文案落位 | Photoshop / Figma / 设计工具 |
| 试读与返修 | 反馈记录、问题标注 | 验证表 / 试读记录表 |

---

## 六、风险点

1. **前置结构不清，后面越做越乱**
2. **图文分工不明，导致重复解释**
3. **视觉锚点不稳，导致风格漂移**
4. **生成过量但筛选标准不清，造成低效**
5. **试读不做记录，导致返修无依据**

---

## 七、一句话总结

AI 协同条漫的核心，不是“更快出图”，
而是把条漫生产变成一条：

**先设计怎么被读，再组织怎么被做出来，最后再用反馈把它修正到成立。**
