# SaltFire 项目开展 SOP v0.1

> 状态：草稿 / 待评审  
> 项目：SaltFire Project - 小红书条漫内容项目  
> 适用阶段：需求文档已对齐，准备进入方案阶段及后续执行阶段

---

## 一、SOP 目标

本 SOP 用于规范 SaltFire 项目从**需求接收 → 方案推进 → 执行协作 → 验收交付 → 复盘沉淀**的完整流程。

核心目标只有三个：
1. 避免项目启动时信息不清、边界混乱
2. 避免执行中任务漂移、责任不清、交接失真
3. 避免交付完成后经验留在聊天里，无法复用

一句话：

> **先对齐，再推进；先落盘，再协作；先交付，再复盘。**

---

## 二、适用范围

本 SOP 适用于 SaltFire 项目下这类任务：
- 内容策划
- 漫画/条漫账号建设
- 连载方案设计
- 角色/世界观衍生内容开发
- 平台化内容实验（如小红书）

不适用于：
- 单次零碎灵感讨论
- 不需要协作的个人即兴草稿
- 已有成熟 SOP 的专门技术任务

## 四、流程总览

```text
1. 需求受理
2. 需求对齐
3. 任务拆解
4. 执行与跟踪
5. 验收与交付
6. 复盘与沉淀
```

---

## 五、需求受理

### 目标
确认“用户到底要什么”，而不是急着进入执行方案。

### 必做动作
1. 记录原始需求
2. 标明项目背景
3. 识别当前阶段（模糊需求 / 需求对齐 / 方案阶段 / 执行阶段）
4. 建立对应目录与文档路径

### 产出物
- 需求对齐稿（如：`00-需求/小红书条漫需求文档-v0.1.md`）

### 判断标准
这一阶段只回答：
- 这件事是什么（what）
- 为什么要做（why）
- 面向谁（who）
- 当前已知什么 / 未知什么

**不要在这一步过早写成执行方案。**

---

## 六、需求对齐

### 目标
把模糊需求转成“大家不会理解歪”的一致对象。

### 必做动作
1. 补齐基础信息
2. 做必要的轻量调研
3. 区分：
   - 已确认信息
   - 基于调研的初步判断
   - 待确认 / 待验证问题
4. 标注文档状态：草稿 / 待验证 / 待评审

### 产出要求
需求文档至少应包含：
1. 原始需求 / 背景
2. 目标 / 目的
3. 当前基础
4. 初步判断（明确标注“基于调研”）
5. 待确认问题

### 验收点
- 是否已经能清楚说出用户想要的是什么
- 是否分清了“已知”和“推定”
- 是否还没滑进具体执行方案

---

## 七、任务拆解

### 目标
将已对齐的需求，转成能执行、能交接、能验收的任务结构。

### 必做动作
1. 指定主任务位（Owner）
2. 指定协助位
3. 明确交付形式
4. 明确时间要求
5. 明确文件路径
6. 明确交接协议

### 推荐输出
- 方案草案
- 任务拆解表
- 里程碑节点

### 交接规则
统一使用 5-field `[HANDOFF]`：
- Where
- What
- How to verify
- Known Issues
- Next Steps

---

## 八、执行与跟踪

### 目标
在执行过程中保持：
- 路径清楚
- 状态可追踪
- 风险能及时暴露

### 执行原则
1. 边界清楚后，执行人立即进入执行，不反复复述需求
2. 所有关键产出优先落盘，再在群里发路径
3. 中途有阻塞，优先报卡点，不要失联
4. 不在群里贴大段上下文，用共享文件系统作为唯一状态真理

### 进度同步最小要求
只允许同步三类信息：
- 开始处理
- 遇到阻塞 / 风险
- 已落盘交付

---

## 九、验收与交付

### 目标
确认交付物是否达到当前阶段标准，而不是“差不多就算了”。

### 必做动作
1. 检查文件路径是否正确
2. read 文件验证实际内容，不只看口头汇报
3. 检查是否满足当前任务的边界与交付要求
4. 如需交接，补齐 `[HANDOFF]`

### 验收结论三分法
- 放行
- 返工
- 暂停

### 交付物归档原则
- 项目文档 → 项目目录
- 可复用方法 → 知识库
- 教训与流程问题 → SOP / memory / knowledge sync

---

## 十、复盘与沉淀

### 目标
让这次项目推进里真正有价值的经验，留下来。

### 必做动作
1. 记录做对了什么
2. 记录哪里失真 / 卡住 / 跑偏
3. 判断哪些经验值得进入知识库或 SOP
4. 必要时通过 `[KNOWLEDGE_SYNC]` 向 Anna 提交

### 复盘最少回答三件事
- 哪一步最容易失真
- 哪一步最容易越位或漏接
- 下一次怎么减少重复错误

## 十二、当前阶段建议（针对 SaltFire 小红书条漫项目）

SaltFire 当前最顺的推进顺序应为：

1. 先完成**需求对齐文档修正**
2. 再转入**方案草案**
3. 方案草案通过后，再拆执行任务
4. 执行过程中严格按文件路径 + handoff 协作
5. 首轮交付后做一次轻复盘，更新 SOP

一句话：

> **现在先别跳着做账号运营细则，先把需求对齐稿和方案草案这两层跑顺。**
