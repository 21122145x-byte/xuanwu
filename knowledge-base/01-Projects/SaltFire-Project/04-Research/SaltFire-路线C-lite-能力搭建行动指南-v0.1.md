# SaltFire-路线C-lite-能力搭建行动指南-v0.1

## 一、文档目的

本指南用于把 SaltFire 路线 C-lite 从“能力缺口判断”，推进到“按阶段搭建的行动顺序”。

目标不是一步到位做出全 Agent 工作室，
而是先把最小闭环搭起来：

**先理解，再结构，再视觉，最后再进入反馈重跑。**

---

## 二、总原则

1. **先搭脑子，再搭手脚**
   - 先做理解层与结构层，再做视觉层与工作台层

2. **先做可跑样本，不做完整系统**
   - 先验证成立性，不先追求工业化

3. **人类先保留判断权**
   - 早期阶段，Agent 负责输出方案，人类负责拍板与筛选

4. **每个阶段都要有明确交付物**
   - 不做“讨论过了算完成”

---

## 三、能力搭建阶段总表

| 阶段 | 阶段目标 | 核心任务 | 交付物 | 完成判定 |
|---|---|---|---|---|
| 阶段1：理解层搭建 | 让 SaltFire 的世界先被稳定读懂 | 搭 Script/Research 输出模板与理解格式 | 术语表 / 世界钩子清单 / 命题重点 / 保留项禁入项 | 能基于正文跑出一版结构化理解结果 |
| 阶段2：结构层搭建 | 让理解结果被压成“入口结构” | 搭 Director 输出模板与阅读结构格式 | 切口定义 / 分段结构表 / 分镜意图 / 尾部钩子 | 能明确第一张样本该如何展开 |
| 阶段3：视觉层搭建 | 让结构能被转成可控视觉方案 | 搭 Visual 输出模板与风格控制方法 | 风格词表 / 视觉锚点表 / Prompt包 / 参数方案 | 能输出一版候选图生成方案 |
| 阶段4：样本层搭建 | 把三层串成一张标准样本 | 形成第一张样本并试读 | 标准样本 v1 / 试读问题单 | 能看到神谕感 / 新秩序 / 阅读入口是否成立 |
| 阶段5：反馈层搭建 | 让错误能被定位和重跑 | 搭 Review 判断规则与重跑规则 | 卡点记录表 / 重跑规则 / 局部修改机制 | 能回答错误发生在理解/结构/视觉哪一层 |

---

## 四、阶段一：理解层搭建

### 4.1 目标
把《盐火预言》中的：
- 术语
- 世界钩子
- 命题重点
- 情绪压强
- 保留项 / 禁入项

先整理成可被后续接住的结构化结果。

### 4.2 要做的事
1. 设计 Script/Research 输出模板
2. 固定术语表格式
3. 固定世界钩子清单格式
4. 固定命题重点提取格式
5. 固定保留项 / 禁入项格式
6. 用真实正文片段跑第一版样例

### 4.3 交付物
- `Script/ResearchAgent 输出模板 v0.1`
- 一次真实正文跑样结果

### 4.4 完成判定
- 能读一段正文并跑出结构化理解结果
- 这份结果可以被 Director 直接拿去继续压结构

---

## 五、阶段二：结构层搭建

### 5.1 目标
把理解层结果压成：
- 第一张样本切口
- 分段节奏
- 分镜意图
- 尾部钩子

### 5.2 要做的事
1. 设计 Director 输出模板
2. 固定切口定义格式
3. 固定分段结构表格式
4. 固定每段分镜意图格式
5. 固定尾部留钩格式
6. 基于阶段一输出，做第一张样本结构稿

### 5.3 交付物
- `DirectorAgent 输出模板 v0.1`
- 第一张标准样本结构稿 v0.1

### 5.4 完成判定
- 能明确“第一眼看什么、中段压什么、结尾留什么”
- 能说清这一张讲什么、不讲什么

---

## 六、阶段三：视觉层搭建

### 6.1 目标
把结构层语言转成：
- 风格方向
- 视觉锚点
- Prompt 包
- 一致性控制方案

### 6.2 要做的事
1. 设计 Visual 输出模板
2. 固定风格关键词表
3. 固定视觉锚点表
4. 固定 Prompt 包模板
5. 固定一致性控制组合规则（LoRA / IP-Adapter / ControlNet）
6. 给第一张样本产出视觉方案

### 6.3 交付物
- `VisualAgent 输出模板 v0.1`
- 第一张样本候选图生成方案 v0.1

### 6.4 完成判定
- 能输出主风格 / 负面约束 / 每段视觉目标 / 参数方案
- 候选图方案能被人工直接拿去测试

---

## 七、阶段四：样本层搭建

### 7.1 目标
把前三层串成真正的第一张标准样本。

### 7.2 要做的事
1. 跑第一张样本切口
2. 生成候选图组
3. 人工筛选
4. 组装长图 / 图文关系
5. 内部试读

### 7.3 交付物
- 第一张标准样本 v1
- 试读问题单

### 7.4 完成判定
- 能判断读者是否看见了“神谕感 / 新秩序 / 新鲜世界入口”
- 能判断是否愿意继续看下一张

---

## 八、阶段五：反馈层搭建

### 8.1 目标
让第一张样本的卡点能被明确定位，并能决定从哪一层重跑。

### 8.2 要做的事
1. 设计反馈记录格式
2. 设计卡点标注方式
3. 设计重跑判断规则
4. 明确哪些问题回到 Script/Research
5. 明确哪些问题回到 Director
6. 明确哪些问题回到 Visual

### 8.3 交付物
- 反馈记录表 v0.1
- 重跑判断规则 v0.1

### 8.4 完成判定
- 能对试读后的问题做层级归因
- 能决定从哪一层重跑，而不是整张推翻

---

## 九、当前不做什么

当前阶段暂不做：
- 全 Agent 自动闭环
- 视频链路
- 多集连续生产
- 工作台完整开发
- 商业化量产
- 自动发布 / 自动导出

这些都属于后续放大阶段，不属于 C-lite 当前任务。

---

## 十、一句话收口

SaltFire 路线 C-lite 的能力搭建，不是从“工具最多的地方”开始，
而是从“最容易把项目带偏的地方”开始。

所以顺序必须是：

**阶段1 理解层 → 阶段2 结构层 → 阶段3 视觉层 → 阶段4 样本层 → 阶段5 反馈层**
