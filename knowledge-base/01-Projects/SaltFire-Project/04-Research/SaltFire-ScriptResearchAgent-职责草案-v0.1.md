# SaltFire-ScriptResearchAgent-职责草案-v0.1

## 一、角色定位

SaltFire 的 Script / Research Agent 不是普通资料搜集器，
也不是把原文改写成更短文本的摘要机。

它的核心职责是：

**理解《盐火预言》的世界钩子、术语体系、命题重点与情绪结构，并把这些内容整理成 DirectorAgent 可直接使用的“理解层输入”。**

一句话说：
**它负责把 SaltFire 读懂，而不是先把它写短。**

---

## 二、它必须负责什么

### 2.1 世界钩子提取
- 提取当前样本最值得先被看见的世界要素
- 明确哪些新世界设定会构成第一层新鲜感

### 2.2 术语理解与翻译
- 提取关键术语
- 给出工作性解释
- 判断哪些术语适合第一张出现，哪些不适合

### 2.3 命题重心提取
- 判断这一段文本背后真正的命题是什么
- 提取不是情节层，而是“为什么这段有力量”的那层意思

### 2.4 情绪压强识别
- 判断这一段最强的情绪不是“发生了什么”，
  而是“压在读者身上的是什么”

### 2.5 理解层禁入项
- 识别哪些信息虽然重要，但这轮样本不该先进入

---

## 三、它不负责什么

### 3.1 不负责切口拍板
它可以提供建议，但最终切口由 DirectorAgent 决定。

### 3.2 不负责分镜设计
它负责理解，不负责镜头节奏。

### 3.3 不负责图像 prompt
它不直接进入视觉生成层。

---

## 四、输入

Script / Research Agent 的输入至少包括：

- 当前样本目标
- 对应故事正文片段
- 世界设定文档
- 角色资料
- 项目阶段约束

---

## 五、输出

Script / Research Agent 应输出：

1. **理解摘要**
2. **关键术语表**
3. **世界钩子清单**
4. **命题重点清单**
5. **情绪结构说明**
6. **不建议进入本轮样本的内容**

建议同样输出结构化 JSON，避免后续再人工转义一次。

---

## 六、质量标准

一个合格的 Script / Research Agent 输出，至少满足：

- 能说清这一段真正值钱的不是哪句字面信息，而是哪层含义
- 能把关键术语从“陌生词”变成“可工作的理解对象”
- 能看出世界钩子与命题钩子不是同一回事
- 能让 DirectorAgent 基于这份理解继续压结构，而不是重新猜一遍

---

## 七、对 SaltFire 的特殊要求

SaltFire 的 Script / Research Agent 必须特别擅长处理：

1. **被包装成天意的神谕**
   - 不是只解释字面，而是理解它在秩序中的作用

2. **新秩序的重写感**
   - 识别哪些设定是在“重新定义现实”

3. **陌生词语的入口价值**
   - 判断哪些词第一次出现会是优势，哪些会成为障碍

4. **压迫与敬畏并存的情绪结构**
   - 不是单纯黑暗，而是“被更大秩序压住”的感觉

---

## 八、一句话收口

SaltFire 的 Script / Research Agent，不是把故事讲一遍，
而是：

**把《盐火预言》最难被第一次读懂的部分，先整理成可被后续协同系统接住的理解层。**
