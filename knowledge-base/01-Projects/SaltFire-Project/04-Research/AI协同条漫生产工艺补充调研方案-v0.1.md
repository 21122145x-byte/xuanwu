# AI协同条漫生产工艺补充调研方案-v0.1

## 一、主调研命题

本轮补充调研的主命题不是零散搜站点，
而是集中回答一个问题：

**Agent 协同技能 / 经验，如何真正参与条漫生产，并为 SaltFire 条漫提供可用的流程支持。**

因此，GitHub、Moltbook、社交网站、技术社区都不是平行大类，
而是围绕这一主命题的不同证据来源面。

---

## 二、调研目标

本轮补充调研要回答 4 个问题：

1. Agent 协同在条漫生产中，已经能参与哪些环节
2. 开源社区与产品社区里，有哪些可复用的工具、脚本、workflow、skill、agent 方案
3. 创作者在真实使用中，积累了哪些可直接借鉴的经验与坑点
4. 对 SaltFire 这种“预言感 / 新秩序 / 暗黑美学”方向，现有 Agent 协同方案是否有适配基础

---

## 三、证据来源面

### 3.1 GitHub（开源方案面）
重点搜：
- repo
- workflow
- script
- tool
- skill
- comic production agent
- image pipeline automation
- storyboard-to-comic workflow

**目的：**
找到可复用、可参考、可直接试跑的开源工作流和自动化脚本。

---

### 3.2 Moltbook / 产品站 / 工具站（产品方案面）
重点搜：
- app
- skill
- agent
- workflow 页面
- 产品能力说明
- 使用案例
- 漫画 / 条漫相关工具页

**目的：**
确认市场上已经成型的产品化方案，以及它们对漫画生产链路的覆盖能力。

---

### 3.3 社交网站创作者实操经验（真实经验面）
重点搜：
- Reddit
- X / Twitter
- YouTube
- Bilibili
- 小红书
- 即刻
- 知乎

重点看：
- 真实复盘
- 失败案例
- 返工记录
- 量产经验
- 一致性解决方案
- Agent / workflow 的真实使用感受

**目的：**
避免只看工具宣传页，补齐“真实创作者怎么做、哪里翻车、哪里真能提效”的一手经验。

---

### 3.4 技术社区 / 教程站（方法沉淀面）
重点搜：
- CSDN
- Hugging Face
- Medium
- Substack
- 优设
- 站酷
- 技术博客

重点看：
- 工作流拆解
- 技术教程
- LoRA / ControlNet / ComfyUI 等进阶用法
- 漫画 / 条漫生产相关技术贴

**目的：**
补足“为什么这样做”的方法论与技术细节。

## 五、执行顺序

### 第一步：GitHub / 开源工作流
先把开源的 agent / workflow / script / tool 拉出来。

### 第二步：Moltbook / 产品站 / 工具站
确认已经产品化的方案有哪些，是否覆盖条漫生产关键链路。

### 第三步：社交平台创作者经验
重点搜真实复盘、失败案例、返工经验。

### 第四步：技术社区补方法细节
用技术贴补齐“为什么这样做、怎么实现一致性与自动化”。

### 第五步：回到 SaltFire 适配性
最后统一判断：现有 Agent 协同经验是否能迁移到 SaltFire 条漫。
