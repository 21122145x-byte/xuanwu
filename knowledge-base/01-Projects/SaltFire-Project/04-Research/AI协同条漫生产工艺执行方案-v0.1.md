# AI协同条漫生产工艺执行方案-v0.1

> 基于调研数据整理，供第五章生产工艺写作参考
> 更新：2026-03-25

---

## 一、生产阶段划分

按调研数据，AI协同条漫生产分为5个阶段：

### 阶段一：视觉锚点确立
**目标：** 确定SaltFire的视觉基调

| 项目 | 内容 |
|------|------|
| 工具 | Midjourney |
| AI动作 | 生成3组概念气氛图（新世界/神谕/天意包装感） |
| 人工动作 | 筛选、确定风格方向（色调/光影/质感） |
| 验收标准 | 视觉基调能传递"预言感"，有辨识度 |

### 阶段二：角色参考建立
**目标：** 建立角色视觉规范，验证LoRA可行性

| 项目 | 内容 |
|------|------|
| 工具 | Flux + ControlNet (OpenPose) + Tensor.art（在线LoRA训练） |
| AI动作 | 生成角色设定表（正/侧/背/表情）；训练角色LoRA |
| 人工动作 | 确定角色视觉规范（发型/服装/特征词） |
| 验收标准 | LoRA训练后角色一致性达标（95%+） |

### 阶段三：分镜脚本转图
**目标：** 把脚本转化为连续分镜图

| 项目 | 内容 |
|------|------|
| 工具 | Stable Diffusion + ComfyUI + LoRA |
| AI动作 | 批量生成各分镜图 |
| 人工动作 | 筛选、局部重绘、节奏调整 |
| 验收标准 | 图文关系成立，节奏顺 |

### 阶段四：排版与后期
**目标：** 完成可发布的成品

| 项目 | 内容 |
|------|------|
| 工具 | 手动排版（或Anifusion） |
| AI动作 | 无 |
| 人工动作 | 添加文字、气泡、整体检查 |
| 验收标准 | 成品可发布，无低级错误 |

### 阶段五：试读验证
**目标：** 获取真实阅读反馈

| 项目 | 内容 |
|------|------|
| 验证动作 | 内部试读/小范围测试 |
| 观察指标 | 能否读完/卡点位置/图文关系评价/风格记忆度 |
| 验收标准 | 能读完+记住画面+愿意继续看 |

## 三、一致性方案（三层系统）

```
Layer 1 - 身份层：LoRA训练
  ↓
Layer 2 - 特征层：精确提示词 + 权重
  ↓
Layer 3 - 构图层：IP-Adapter / ControlNet
```

推荐数据集准备：
- 15-30张高质量角色图
- 70%面部特写 + 30%全身
- 不同角度/姿势/表情
- 训练时间：1-4小时

## 四、必须避开的坑

| 坑点 | 表现 | 解决方案 |
|------|------|---------|
| 画风漂移 | 第3页日漫、第5页变欧美卡通 | 固定风格关键词、写实描述词 |
| 角色换脸 | 长发变短发、五官变化 | 训练LoRA、建角色关键词库 |
| 脚本画面脱节 | 文字悲伤、画面无表情 | 脚本附情绪标注 |
| 背景不一致 | 同一场景背景乱变 | ControlNet锁定背景 |

## 五、SaltFire专项风险

| 风险 | 说明 | 建议 |
|------|------|------|
| 暗黑美学适配 | 现有案例多偏日漫/韩风，预言感/天意包装风格无直接参考 | 阶段一专门测试，失败则调整风格方向 |
| 神谕感视觉化 | "被制造的天意"如何视觉呈现 | 调研阶段无答案，需实测 |

## 七、下一步动作

1. **阶段一测试**：用Midjourney生成3组SaltFire风格概念图，验证暗黑美学/预言感是否可达
