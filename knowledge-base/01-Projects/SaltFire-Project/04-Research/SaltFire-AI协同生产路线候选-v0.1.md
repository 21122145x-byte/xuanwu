# SaltFire-AI协同生产路线候选-v0.1

## 一、目的

本文件用于在补充调研完成后，给 SaltFire 条漫项目提供可直接进入样本测试的路线候选。

目标不是列出所有可能，
而是压出 3 条最可执行的路线，供后续拍板。

---

## 二、路线 A：轻量样本验证型

### 核心思路
先不追求全 Agent 化，
而是以最小可行链路快速做出一张标准样本，验证“预言感是否能成立”。

### 基本链路
- 人工确定切口
- 人工拆阅读结构
- Midjourney / SD 做概念图与主画面
- LoRA / IP-Adapter 做角色与风格一致性控制
- 手动排版与试读

### 优点
- 成本最低
- 上手最快
- 最适合做第一张样本
- 能最快验证 SaltFire 的核心视觉方向

### 风险
- 自动化程度低
- 复用性一般
- 后续放大时还要重搭流程

### 适用判断
如果目标是：
**最快做出第一张能判断成不成立的样本，优先选这条。**

---

## 三、路线 B：角色一致性优先型

### 核心思路
先把角色、风格与画面一致性这条链打稳，
再去推进更复杂的条漫连续生产。

### 基本链路
- 人工确定切口与角色视觉规范
- Flux / SD + ControlNet 生成角色设定图
- 训练角色 LoRA
- ComfyUI 建立稳定生成 workflow
- 按段落逐步出图
- 手动排版与试读

### 优点
- 最有利于解决“换脸 / 风格漂移”问题
- 后续更适合进入批量样本生产
- 对长线连载更友好

### 风险
- 前期准备时间更长
- 样本速度慢于路线 A
- 如果第一张样本就失败，前期投入较重

### 适用判断
如果目标是：
**把 SaltFire 当成可持续系列项目来准备，而不是只做一次验证，优先考虑这条。**

---

## 四、路线 C：多 Agent 协作实验型

### 核心思路
把 SaltFire 最复杂的部分——脚本理解、术语理解、氛围理解——交给 Agent 协作，
图像生成只承担视觉呈现。

### 基本链路
- Research / Script Agent 负责拆世界观、术语、命题重点
- Director Agent 负责阅读结构与镜头规划
- Visual Agent 负责提示词与图像生成控制
- Dialogue / Layout Agent 负责文字与气泡
- 人类只做方向拍板与最终筛选

### 优点
- 最贴合 SaltFire 这种高概念、高术语密度项目
- 有机会把“理解层”和“呈现层”真正解耦
- 如果跑通，后续扩展空间最大

### 风险
- 系统复杂度最高
- 前期搭建成本高
- 很可能在第一轮就暴露一堆协同问题
- 容易为了跑流程而牺牲样本速度

### 适用判断
如果目标是：
**验证 SaltFire 是否适合走高协同、类工作室化的 AI 生产路线，可以把这条作为实验路线。**

---

## 五、当前推荐顺序

### 推荐顺序：A → B → C

原因：

1. **先用 A 验证方向成立**
   - 确认神谕感 / 新秩序 / 情绪张力能不能被视觉化

2. **再用 B 稳定一致性问题**
   - 把角色与风格统一做扎实

3. **最后视情况尝试 C**
   - 如果 SaltFire 真的需要高协同理解层，再投入多 Agent 实验

---

## 六、当前建议

对 SaltFire 来说，当前最稳的策略不是直接冲最复杂路线，
而是：

> **先用路线 A 做第一张标准样本，验证“这个世界能不能被看见”；如果成立，再过渡到路线 B 稳定生产；路线 C 作为后续增强路线。**

---

## 七、一句话收口

现在不是“选最先进的路线”，
而是选：

**哪条路线最能让 SaltFire 先成立。**
