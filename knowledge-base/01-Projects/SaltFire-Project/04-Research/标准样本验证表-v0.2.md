# 标准样本验证表-v0.1

## 一、验证目的

本表用于从“行业数据验证”切入“样本实测验证”。

目标不是继续讨论概念是否说得通，
而是通过一张标准样本，验证以下问题：

1. 条漫内容是否能被顺利读下去
2. 图文关系是否成立
3. 视觉风格与气质是否能被记住
4. 风格一致性方案是否可控
5. 当前产品定义中的关键推导，是否能被一手反馈支撑

---

## 二、验证原则

- 先做一张标准样本，不先铺开多张内容
- 先验证成立性，不先追求规模化
- 先看阅读反馈，再看审美偏好
- 所有判断尽量落到可观察信号，不只靠主观感觉
- 记录正反馈，也记录卡点与流失点

## 四、验证板块总表

| 验证板块 | 核心问题 | 观察点 | 验证方式 | 通过信号 | 风险信号 |
|---|---|---|---|---|---|
| 阅读成立性 | 读者能不能顺利往下读 | 是否滑完、是否中途卡住、哪里退出 | 内部试读 / 小范围试读 | 能顺利读完并复述大意 | 中途断裂、看不懂在讲什么 |
| 图文关系 | 图和字是否真正共同叙事 | 是否重复解释、是否互相打架、是否信息失衡 | 试读反馈 + 主创复盘 | 图文互补，信息负担适中 | 图说一遍字再说一遍，或两边都不够 |
| 风格气质 | 视觉风格是否有辨识度 | 是否能留下画面印象、气质是否统一 | 试读后回忆反馈 | 能记住一个画面/一种气质 | 看完无记忆点、风格漂移 |
| 节奏设计 | 滑动中的停顿与推进是否成立 | 哪些段落拖、哪些段落太快、结尾是否有钩 | 试读记录 + 复盘 | 节奏顺、能自然停留 | 留白无效、推进混乱、尾部泄气 |
| 风格一致性 | 多图之间是否像同一作品 | 角色稳定性、场景语言、色调一致 | 视觉检查 + 返工记录 | 风格统一，返工可控 | 角色漂、画风散、返工成本高 |

## 五、重点验证指标

### 5.1 阅读反馈指标

| 指标 | 记录方式 | 备注 |
|---|---|---|
| 是否完整读完 | 是 / 否 | 最基础成立性 |
| 卡顿位置 | 标注具体段落 | 用于找节奏问题 |
| 能否复述大意 | 是 / 否 / 部分 | 验证理解度 |
| 是否愿意继续看下一篇 | 是 / 否 | 验证续读潜力 |

### 5.2 内容反馈指标

| 指标 | 记录方式 | 备注 |
|---|---|---|
| 记住了什么 | 画面 / 台词 / 概念 / 情绪 | 验证记忆点 |
| 看不懂的地方 | 具体记录 | 验证信息密度 |
| 图文关系评价 | 顺 / 重 / 乱 / 失衡 | 验证图文协同 |
| 节奏评价 | 顺 / 拖 / 跳 / 泄 | 验证结构设计 |

### 5.3 生产反馈指标

| 指标 | 记录方式 | 备注 |
|---|---|---|
| 风格一致性 | 高 / 中 / 低 | 验证视觉稳定性 |
| 返工次数 | 数字记录 | 验证流程可控性 |
| 高成本环节 | 具体标注 | 找生产瓶颈 |
| AI辅助有效环节 | 具体标注 | 找提效点 |

## 七、完成判定

本轮样本验证完成，至少满足以下条件：

- 有一份完整样本成品
- 有至少一轮试读记录
- 能明确指出最强反馈与最大问题
- 能判断当前产品定义中哪些推导被支撑、哪些被打脸
- 能决定是继续优化样本，还是进入下一轮生产

---

## 八、使用说明

本表用于标准样本验证，
作用是把“概念推导”推进到“真实反馈”。

它不替代正式方案，
也不替代长期数据监测，
而是帮助团队回答：

**这套条漫产品，到底有没有在真实阅读里成立。**
