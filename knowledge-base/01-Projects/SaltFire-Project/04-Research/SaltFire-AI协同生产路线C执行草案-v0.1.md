# SaltFire-AI协同生产路线C执行草案-v0.1

## 一、路线定位

路线 C 的目标，不是直接搭一个“全自动工作室”，
而是先验证：

**多 Agent 协同是否真的能帮助 SaltFire 这类高概念、高术语密度、高情绪张力项目，把“理解层”稳定转译为“可见层”。**

因此，路线 C 当前采用 **C-lite** 起步：
- 先上最小 3 Agent 协同
- 人类保留最终拍板与最终筛选
- 先做一张标准样本
- 不追求一上来跑完整系列生产

---

## 二、路线 C 的最小目标

第一轮只验证 3 件事：

1. Agent 是否能正确理解《盐火预言》的世界钩子、术语与命题重点
2. Agent 组织出来的切口与阅读结构，是否比纯人工直觉更稳定
3. 视觉生成层是否能在这种协同结构下，更好地保住 SaltFire 的“神谕感 / 新秩序 / 情绪压强”

一句话说：
**先验证 C 能不能帮 SaltFire“理解得更准、转译得更稳”。**

---

## 三、三核心 Agent 结构

### 3.1 Script / Research Agent

**职责：**
- 读取与理解《盐火预言》的相关文本材料
- 提取当前样本需要保留的世界钩子、术语、命题重点
- 输出“不可丢失内容”与“禁入内容”

**输入：**
- 故事正文 / 世界设定 / 角色资料
- 当前样本目标

**输出：**
- 样本理解摘要
- 关键术语解释表
- 核心命题点清单
- 禁入清单

**不负责：**
- 直接出图
- 决定最终视觉风格

---

### 3.2 Director Agent

**职责：**
- 将 Script / Research Agent 的理解结果压成条漫切口
- 设计阅读结构、镜头意图、分段节奏
- 确定“这一张先让读者看到什么”

**输入：**
- 样本理解摘要
- 核心命题点清单
- 禁入清单

**输出：**
- 切口定义句
- 分段结构表
- 分镜意图说明
- 图文分工建议

**不负责：**
- 直接决定模型参数
- 直接定稿画面

---

### 3.3 Visual Agent

**职责：**
- 把 Director Agent 输出的结构转成视觉生成方案
- 组织 prompt、风格词、参考图与控制参数
- 产出第一轮候选图组

**输入：**
- 分段结构表
- 分镜意图说明
- 图文分工建议
- 风格方向要求

**输出：**
- Prompt 包
- 参数表
- 候选图组
- 每段可选视觉方案说明

**不负责：**
- 解释剧情逻辑
- 取代最终人工筛选

---

## 四、人类保留的决策点

路线 C-lite 中，人类不退出，保留两个核心位置：

### 4.1 方向拍板
负责判断：
- 当前切口是否真的是 SaltFire 的“第一入口”
- 哪些内容必须保留，哪些不能贪多
- Agent 的理解有没有跑偏

### 4.2 最终筛选
负责判断：
- 候选图是否真正有 SaltFire 的气质
- 哪个版本最能承接“神谕感 / 新秩序 / 情绪压强”
- 画面是否只是“对”，还是“有魂”

一句话：
**Agent 负责理解与组织，人类负责判断与定魂。**

---

## 五、第一张标准样本的 C-lite 工作流

### 第 1 步：样本目标定义
- 明确第一张样本要验证什么
- 默认不是“讲清楚故事”，而是“打开世界入口”

### 第 2 步：Script / Research Agent 理解材料
- 输入故事文本与设定资料
- 输出：术语、命题点、世界钩子、禁入内容

### 第 3 步：Director Agent 压缩结构
- 生成切口定义
- 生成分段结构
- 明确开头钩子 / 停顿 / 结尾留钩

### 第 4 步：Visual Agent 组织视觉生成
- 生成 prompt 与参数
- 按段落逐段出候选图
- 提供多套视觉方案

### 第 5 步：人工初筛
- 选出最接近 SaltFire 气质的候选
- 淘汰“只是好看但不对味”的版本

### 第 6 步：局部修正与排版
- 调整一致性、图文关系、长图节奏
- 做成标准样本 v1

### 第 7 步：试读与反馈
- 检查是否读得下去
- 是否看见了神谕 / 新秩序 / 压迫感
- 是否愿意继续往下看

---

## 六、第一轮只验证什么，不验证什么

### 本轮只验证
1. Agent 对 SaltFire 术语与世界钩子的理解是否靠谱
2. Agent 压出来的结构是否更稳
3. 视觉生成是否能更准确承接 SaltFire 的气质
4. 三 Agent 协作是否真有收益

### 本轮不验证
1. 完整系列生产效率
2. 全自动发布链路
3. 长线角色库管理
4. 多集连续性控制
5. 商业化量产能力

也就是说：
**第一轮只验证“这条路能不能成立”，不验证“它以后能不能开工厂”。**

---

## 七、路线 C 的风险点

1. **Agent 理解层跑偏**
   - 术语理解对了，但命题重心抓错

2. **系统复杂度过高**
   - 花太多时间搭系统，反而拖慢样本产出

3. **视觉层看起来对，但没有 SaltFire 的魂**
   - 图像正确，不等于气质成立

4. **协同收益不明显**
   - 如果三 Agent 没有比纯人工更稳，那路线 C 就不能轻易放大

---

## 八、当前建议

路线 C 可以启动，
但必须以 **C-lite** 起步。

当前最合理的动作不是追求全自动闭环，
而是：

> **先用 3 个核心 Agent 做出第一张标准样本，验证 SaltFire 最难的“理解层 → 可见层”转译是否成立。**

---

## 九、一句话收口

路线 C 不是因为它最先进才被选中，
而是因为 SaltFire 最难的部分，恰好需要一条“多层理解 → 多层转译”的协同路线。
