# 🤖 AI写作小组架构方案 — 盐火写作工坊

## 一、小组定位

**名称**：盐火写作工坊  
**使命**：持续产出高质量虚构作品，建立可复用的AI辅助写作方法论  
**核心循环**：学习 → 沉淀 → 输出 → 收集反馈 → 迭代输出

## 三、工作流程（SOP）

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  学习   │ → │  沉淀   │ → │  输出   │ → │  反馈   │ → │  迭代   │
│  (粉雪) │    │  (富贵) │    │  (旺财) │    │  (富贵) │    │  (旺财) │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↑___________________________________________________________↓
```

### 阶段详解

| 阶段 | 负责人 | 交付物 | 时限 |
|------|--------|--------|------|
| **学习** | 粉雪 | 参考资料包、市场情报、竞品分析 | 按需 |
| **沉淀** | 富贵 | 故事大纲、人物小传、节奏曲线、世界观文档 | 2-3天 |
| **输出** | 旺财 | 初稿（按章节） | 每章1-2天 |
| **反馈** | 富贵+外部 | 修改意见、读者反馈、问题清单 | 持续 |
| **迭代** | 旺财 | 修订稿 | 每轮1-2天 |

## 四、知识管理系统

### 文档结构
```
knowledge-base/
└── 01-Projects/
    └── SaltFire-Project/           # 《盐火预言》专项
        ├── 01-Story/
        │   ├── Outline.md          # 故事大纲
        │   ├── Characters/         # 人物档案
        │   ├── Worldbuilding/      # 世界观设定
        │   └── Chapters/           # 分章文档
        ├── 02-Process/
        │   ├── Feedback/           # 读者反馈归档
        │   ├── Revisions/          # 修改记录
        │   └── Lessons.md          # 经验教训
        ├── 03-Reference/
        │   ├── Research/           # 粉雪搜集的资料
        │   └── Competitors/        # 竞品分析
        └── 04-Output/
            ├── Drafts/             # 初稿
            ├── Final/              # 定稿
            └── Blinded-Tests/      # 盲测材料
```

### 经验沉淀模板（Anna负责）

每次项目结束，输出：
- **复盘文档**：什么对了、什么错了、为什么
- **方法论更新**：可复用的技巧、避免的坑
- **知识库补充**：新的人物模型、节奏技巧、世界观组件

## 五、反馈收集机制

### 内部反馈（富贵主导）
- 每章完成后，富贵审阅并出修改意见
- 使用结构化反馈模板（人物/节奏/设定/文笔四维度）

### 外部盲测（Shaun负责，粉雪协助）
- Shaun负责找测试读者、发放问卷
- 粉雪协助整理反馈数据
- 使用标准化盲测清单（抓力/人味/节奏/设定负担）
- 反馈归档到 `02-Process/Feedback/`

### 反馈处理流程
```
收集 → 分类（人物/节奏/设定/文笔）→ 评估（普遍问题vs个别口味）
  → 优先级排序 → 制定修改方案 → 执行修改 → 验证效果
```
