## 背景/目的

### 背景
SaltFire 是一个原创架空危机故事，核心是"预言如何被包装成天意，再被用来推动资源重分配、市场恐慌、军事动员和秩序重写"。世界建立在"炽盐"能源与"天幔"预言网络之上。

现有前三章正文、世界框架、角色卡等可继续开发的文本基础。

### 目的
在小红书建立条漫账号，通过条漫/连载故事形式发布 SaltFire 内容，实现引流目标。

## 内容形态确认 ✅

### 条漫定义（需求轮工作定义）

条漫是以**竖版连续图片序列**为基本表达单位，图文结合讲述故事的叙事形式。

**条漫成立的最低条件：**
1. 竖版布局（适配手机竖屏阅读）
2. 图像与文字共同叙事
3. 叙事连贯性（各格之间有故事推进关系）
4. 视觉吸引力（单格/单图能在滑动瞬间抓住注意力）

**条漫不是什么（排除项）：**
| 形态 | 核心区别 |
|------|----------|
| 插画 | 单张视觉作品，不承担叙事连续性 |
| 分镜 | 制作脚本，非最终成品形态 |
| 漫画 | 通常横向/多格排版，非竖屏优先 |
| 短视频脚本 | 依赖时间轴，条漫依赖空间轴 |

---

### 《盐火预言》改编为条漫的可行性判断

**✅ 改编可行。**

| 保留项 | 风险项 |
|--------|--------|
| 世界观框架（炽盐能源、天幔预言网络） | 需建立"视觉符号系统"将"天意包装"概念视觉化 |
| 核心冲突（资源重分配、市场恐慌、军事动员、秩序重写） | 文字描写的细腻心理层次需转化为画面暗示 |
| 角色关系与权力结构 | 多线叙事交叉受条漫线性节奏限制 |
| 叙事深度与情感张力 | 文字独有的"预言文体"质感需通过视觉符号重新表达 |

## 平台适配确认 ✅

### 小红书承接能力

**结论：小红书可以稳定承接条漫内容。**

- 条漫是小红书主流内容形态之一，有稳定消费群体
- 算法基于用户行为（点赞/收藏/分享）分发，对深度叙事内容有适配性
- 平台有连载内容消费习惯，读者愿意追更

**关键区分："能发"≠"能成立"。** 需确认的是：《盐火预言》这类深度叙事条漫能否被目标受众找到并形成追更。

### 平台角色定位

**主平台（测试验证为主）**

理由：小红书是可行的启动平台，但《盐火预言》的深度叙事属性决定了它不适合完全依赖平台自然流量，需要主动引流路径配合。

## 范围边界

### 包含
- 原创条漫连载
- 有叙事深度的故事内容
- 与 SaltFire 世界观相关的故事解读/延展内容

### 不包含
- 纯搞笑/日常向内容
- 与 SaltFire 世界观无关的主题
- 政治敏感内容（时政讽刺类）

### 待在方案轮处理（标注为风险）
- 引流路径的设计
- "视觉符号系统"的建立方案
- 深度叙事条漫的实际自然流量表现（需实测验证）
