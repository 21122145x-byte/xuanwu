# zopia-skill 能力边界与正确使用说明

- 日期：2026-04-05
- 对象：当前版本 `zopia-skill`
- 当前结论：**zopia-skill 更像“项目制视频制作代理前端”，不是“镜头级精修工具”。**

---

## 一、先把一句话说死

对当前版本 zopia-skill，最准确的口径是：

> **你把故事 / 分镜 / 画面要求交给它，配好项目级参数，它负责在后端把整条短片流水线往下跑。**

它更像：
- 项目管理器
- 参数配置器
- 需求传达器
- 进度轮询器
- 结果取件器

而不是：
- 单镜头编辑器
- 分镜参数 patch 工具
- 素材上传器
- 本地图片首帧绑定器
- 镜头级视频后处理器

一句话：

> **它是“整片生成器”，不是“镜头手术刀”。**

---

## 二、它擅长做什么

根据当前 skill 设计和实际试跑，zopia-skill 擅长的是：

### 1. 建立项目制创作流程
- 创建项目
- 管理剧集
- 设置风格 / 比例 / 模型 / 分辨率

### 2. 把用户原始需求交给后端 Agent
- 发送创作请求
- 让后端 Agent 自行理解并推进“剧本 → 角色 → 分镜 → 视频”的流程

### 3. 追踪进度与取回结果
- 轮询 session
- 查看 workspace 快照
- 下载已生成资产

### 4. 适合从零开始跑完整任务
比如：
- 文生视频
- 文生角色
- 文生分镜
- 从一个完整创作需求出发生成整条短片

一句话：

> **它最适合拿来跑“完整项目”，而不是修“单个镜头”。**

---

## 三、它不擅长做什么

当前版本 zopia-skill 明确不擅长，或当前未提供能力的，包括：

### 1. 精修单条 shot / 分镜参数
当前没有看到接口支持：
- 单独修改某条 shot 的时长
- patch 某个分镜的视频参数
- 对失败 shot 做参数级重跑

### 2. 导入外部 / 本地图片作为 start_frame
当前试跑已验证：
- 把外部图片 CDN 链接直接作为 start_frame 参考重新提交，不闭环
- skill 当前也没有本地图片上传进项目的接口

### 3. 像剪辑台一样逐镜头微调
它不是用来：
- 手动修镜头时长
- 手动换某条镜头首帧
- 手动给单镜头做细颗粒 patch

### 4. 当前前台能力不能代表平台全部能力
当前 skill 不能做，不等于平台绝对不能做；
但至少说明：

> **当前这版 skill 并不提供这些前台手动介入能力。**

---

## 四、为什么前面容易误判

前面的误判，核心在于把 zopia-skill 理解成了：

> “一个可精修、可补参数、可手动修单镜头的视频工具”。

但重新读 skill 之后可以确认：

### 它真正的设计哲学是
- 你负责：配置、传话、取件
- 后端 Agent 负责：理解创作、推进流程、生成资产

也就是说：
- 不建议前端过度拆 shot
- 不建议前端手动重写复杂分镜逻辑
- 不建议前端强行把它当成镜头 patch 工具

一句话：

> **这套 skill 的预设是“少动手，多交给后端 Agent 跑完整流程”。**

---

## 五、这会怎么改变后续使用方式

### 正确使用方式
后续使用 zopia-skill，应该优先采用：

#### 方式 A：完整创作任务直发
把更完整、更清楚的创作任务交给后端 Agent，让它自己决定：
- 分镜怎么拆
- 角色怎么长
- 视频怎么接

#### 方式 B：只在项目级设置上调参
优先调：
- 风格
- 比例
- 视频模型
- generation_method
- 分辨率

#### 方式 C：把它当结果链路，而不是镜头工具链
用它来：
- 跑项目
- 看进度
- 取结果

而不是用它来：
- 修 shot
- patch 单个镜头
- 管素材导入

---

## 六、当前试跑带来的正式经验

通过“盐火预言”试跑，当前已经可以形成两条正式经验：

### 经验 1
zopia-skill 可以用于：
- 新建项目
- 跑通图像生成
- 推进视频生成流程
- 返回明确进度和错误信息

### 经验 2
当前 zopia-skill 不适合用于：
- 解决某条 shot 的底层时长不兼容问题
- 把外部图片重新导入新项目做 start_frame
- 当作镜头级精修工具来救单条视频闭环

---

## 七、后续如果继续推进，应该怎么分工

### 用 zopia-skill 做的
- 从完整创作意图出发创建项目
- 配置合适模型和参数
- 把需求原话交给后端 Agent
- 跟进结果

### 不要强塞给 zopia-skill 做的
- 单条镜头参数修补
- 外部素材导入
- shot 级 patch
- 视频后处理或重挂载

### 如果必须做这些
应改走：
- 平台原生 Web 能力
- 新增 skill 脚本
- 或补平台 API 接口

---

## 八、一句话结论

> 当前 zopia-skill 的正确理解，不是“我拿它做一个可精修的图生视频工具”，而是“我把一套相对完整的创作意图交给它，它帮我跑整条短片生产流程”；因此，后续应把它当作项目制创作代理前端来用，而不是镜头级 patch 工具。
