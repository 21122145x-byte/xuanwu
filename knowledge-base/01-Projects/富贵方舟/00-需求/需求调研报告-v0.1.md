## 1. What（做什么）

### 市场上常见的个股分析交付形式

| 形式 | 代表产品 | 说明 |
|------|----------|------|
| **交互图表型** | TradingView、Finviz | 可点击查看历史数据，模块化展示 |
| **图文报告型** | 东方财富、同花顺 | 图表 + 文字说明，可视化程度高 |
| **文字报告型** | Newsletter、券商研报 | 纯文字 + 表格，结构化文字输出 |
| **视频/音频型** | 少数付费服务 | 较少见，成本高 |

### 富贵方舟现有的方案形式
- 纯文字报告（Markdown 格式的日报/周报）
- 表格化数据展示

---

## 2. Why（为什么）

### 为什么 format 很重要

| 原因 | 说明 |
|------|------|
| **阅读效率** | 图文 > 纯文字，数据可视化降低认知负担 |
| **信息密度** | 图表能同时展示多维度数据 |
| **决策支持** | 直观图表比数字表格更容易判断趋势 |
| **情感体验** | 好的可视化提升使用意愿 |

### 为什么不选更复杂的 format

| 原因 | 说明 |
|------|------|
| **开发成本** | 交互图表需要前端开发，复杂度高 |
| **数据依赖** | 可视化需要稳定的数据源 |
| **维护成本** | 图表比文字更容易因数据变动出错 |
| **推送难度** | 飞书对复杂图表支持有限 |

## 4. How（怎么做）

### 方向性路径

| 路径 | 优缺点 |
|------|--------|
| **A. 纯文字优先** | 简单、快、飞书友好；但信息密度低 |
| **B. 图文结合** | 信息密度高、体验好；但开发成本高，飞书推送有限制 |
| **C. 混合路径** | 核心内容文字，附截图/图表；平衡效率与体验 |

### 推荐路径
**C. 混合路径**：日常推送用文字+表格，深度报告用图文（截图或外链）

## 6. Format（交付形式）

### 各交付物的 format

| 交付物 | Format | 说明 |
|--------|--------|------|
| **个股日报** | 文字 + 表格 | 飞书消息推送，简洁高效 |
| **个股周报** | 文字 + 表格 + 截图 | 更详细，附图表截图 |
| **事件快评** | 纯文字 | 快速响应，简短 |
| **政策解读** | 文字 | 结构化分析 |
| **深度分析** | 图文（外链） | 可做成独立文档或飞书多维表格 |

### Format 决策建议

| 场景 | 推荐 Format |
|------|-------------|
| 每日推送 | 纯文字 + 表格（飞书原生） |
| 每周深度 | 文字 + 表格 + 图表截图（可外链） |
| 实时事件 | 纯文字 |
| 交互需求 | 未来可考虑飞书多维表格 |

---

## 7. 竞品 Format 对比

| 产品 | Format | 优势 | 劣势 |
|------|--------|------|------|
| Finviz | 交互图表 | 信息全面、可筛选 | 复杂度高，不适合推送 |
| TradingView | 交互图表 | 图表强大、社区丰富 | 需要主动访问 |
| 东方财富 | 图文报告 | 可视化好、品牌可信 | 商业产品、不可定制 |
| 同花顺 | 图文报告 | 用户基数大 | 付费功能多 |
| 富贵方舟（现有方案） | 纯文字 | 简单、可推送 | 信息密度低 |

---

## 8. 结论与建议

### Format 建议

**短期（P0）：**
- 日报/周报：文字 + 表格
- 事件快评：纯文字
- 飞书原生推送

**中期（P1）：**
- 周报增加图表截图（手动或自动化）
- 考虑飞书多维表格承载结构化数据

**长期（P2）：**
- 若有交互需求，可开发独立网页/应用
- 图文报告可外链到独立文档

### 核心判断

> **纯文字推送 + 表格** 是目前最可行、最高效的方案，能够满足日常跟踪需求，且技术实现简单、推送成功率高。
>
> **图文/图表** 作为辅助，不作为主要交付形式。
