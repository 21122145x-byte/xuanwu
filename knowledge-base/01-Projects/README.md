# 01-Projects

> 团队项目层。承载具体项目的方案、流程、样例、数据源、阶段推进记录与项目专属执行材料。

---

## 这一层放什么
- 具体项目页 / 项目 README
- 项目方案、定位、范围定义
- 项目专属流程、协作规范、检查清单
- 项目样例、数据源、素材清单
- 项目阶段推进记录
- 项目专属 Subagent / 执行材料

## 这一层不放什么
- 公共制度总纲、团队宪法 → 去 `01-Constitution/`
- 公共方法论、可复用系统设计 → 去 `02-Areas/`
- 纯参考资料、文章、SOP 收藏 → 去 `03-Resources/`
- 临时收件、未判断去向材料 → 先去 `00-Inbox/`
- 团队索引、状态盘点、维护规则 → 去 `99-Meta/`

---

## Active / Archive 使用口径

### Active
满足任一情况，可视为 Active：
- 当前仍在推进
- 最近仍有新增文件 / 明确负责人
- 仍需要继续产出、验证或维护

### Archive
满足以下情况之一，可转 Archive：
- 项目已结束且近期不再推进
- 仅保留历史记录与复盘价值
- 已有稳定产物，不再作为当前工作主线

> 当前 `Archive/` 仍基本空置，说明项目生命周期管理尚未真正跑起来。后续应逐步启用。
