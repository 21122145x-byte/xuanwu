## 一、任务目标

### 1.1 核心目标
生成每日 A 股收盘财经日报，包含：
- 三大指数（沪指、深成指、创业板指）收盘数据
- 涨跌家数统计
- 板块热点（领涨/领跌）
- 市场解读与风险提示

### 1.2 扩展目标（Phase 2-4）
- 数据可视化图表
- 板块资金流向
- 个股异动监测
- 情绪指标分析

---

## 二、触发方式

### 2.1 当前阶段：手动试跑
- **触发条件**：Shaun 手动 @富贵 触发
- **目的**：验证 skill 模块稳定性，积累成功执行记录
- **成功标准**：连续 3 次手动触发成功，无 P0/P1 失败

### 2.2 恢复 cron 条件
- 连续 3 次手动试跑成功
- Anna 审计通过
- 粉雪确认无异常

---

## 三、输入源

### 3.1 主数据源
- **AkShare**（Python 库）
- 覆盖：指数行情、个股数据、板块数据

### 3.2 备用数据源
- **Playwright 爬虫**：东方财富、新浪财经
- 触发条件：AkShare 数据缺失或异常

### 3.3 降级策略
- P0 数据（三大指数、涨跌家数）必须获取
- P1 数据（板块数据）缺失时标记"待补充"
- P2 数据（情绪指标）缺失时可跳过

---

## 四、处理流程（S1-S7 + J1-J4）

### 4.1 Skill 化步骤（S1-S7）

| 步骤 | 名称 | 功能 | 输出 | 失败处理 |
|------|------|------|------|----------|
| S1 | 数据拉取 | 从 AkShare/爬虫获取原始数据 | raw_data.json | 切换备用源 |
| S2 | 数据清洗 | 格式化、去重、校验 | clean_data.json | 人工介入 |
| S3 | 数据校验 | 检查数据完整性、合理性 | validation_report.json | 标记异常 |
| S4 | 内容生成 | 生成市场解读文字 | draft.md | 降级为模板 |
| S5 | 内容优化 | 润色、格式调整 | polished.md | 跳过 |
| S6 | 可视化生成 | 生成图表（Phase 2+） | charts.png | 降级为纯文本 |
| S7 | 文件输出 | 保存到知识库 | final.md | 重试 |

### 4.2 人工判断步骤（J1-J4）

| 步骤 | 名称 | 执行者 | 判断内容 |
|------|------|--------|----------|
| J1 | 数据源选择 | 富贵 | 主源/备用源/降级 |
| J2 | 异常处理决策 | 富贵 | 重试/跳过/人工介入 |
| J3 | 内容质量确认 | 富贵 | 发布/打回修改 |
| J4a | 数据校验 | 数据校验 Agent | 数据完整性检查 |
| J4b | 审计放行 | Anna（人工） | 最终放行/打回/升级 |

**重要说明**：J4b 是团队审计位职责，**不 skill 化**，由 Anna 人工执行。

## 五、Spawn 机制

### 5.1 设计原则
- **局部 spawn**：仅在具体步骤需要时 spawn 子 Agent
- **主链富贵控**：富贵保持主控，不全程托管

### 5.2 Spawn 场景

| 场景 | Spawn 对象 | 输入 | 输出 |
|------|-----------|------|------|
| S2 数据清洗 | 数据清洗 Agent | raw_data.json | clean_data.json |
| S4 内容生成 | 写稿 Agent | clean_data.json | draft.md |
| S5 内容优化 | 旺财 | draft.md | polished.md |
| J4a 数据校验 | 数据校验 Agent | clean_data.json | validation_report.json |

### 5.3 Agent 模板状态

| Agent | 状态 | 提供者 |
|-------|------|--------|
| 数据拉取 Agent | ✅ 已完成 | 富贵 |
| 数据清洗 Agent | ✅ 已完成 | 富贵 |
| 写稿 Agent | ✅ 已完成 | 旺财 |
| 数据校验 Agent | ⏳ 待开发 | 富贵 |

## 八、完成标准

### 8.1 成功标准
- P0 数据完整获取
- 内容生成符合模板
- 文件成功保存到知识库
- 无 P0/P1 失败

### 8.2 失败分类

| 级别 | 定义 | 处理 |
|------|------|------|
| P0 | 核心数据缺失，无法生成日报 | 立即停止，人工介入 |
| P1 | 部分数据缺失，可降级输出 | 标记缺失，继续执行 |
| P2 | 非核心功能异常 | 记录日志，跳过 |

---

## 九、升级路径

### 9.1 失败升级
- P0 失败 → 立即停止 → 人工介入 → 修复后重试
- 连续 3 次 P0 失败 → 暂停任务 → 团队会诊

### 9.2 功能升级
- Phase 1 稳定 → 进入 Phase 2 可视化
- 每 Phase 需 Anna 审计通过
