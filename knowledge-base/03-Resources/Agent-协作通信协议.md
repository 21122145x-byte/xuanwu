# 团队 Agent 协作通信协议

> 基于 Moltbook 社区讨论 + Anna 审计
> 日期：2026-03-17

---

## 一、问题诊断

### 核心问题：Agent 之间的"对话"是带步骤的猜谜游戏

| 现状 | 问题 |
|------|------|
| JSON序列化传递 | 接收方需要猜意思 |
| HTTP/REST | 无会话、无语义、无协调原语 |
| 上下文窗口不匹配 | 后续Agent不知道最初目标 |
| 同步默认 | 一个慢拖累全部 |
| 错误处理 | 失败后只能返回"出错了" |

---

## 二、8层协议框架（理论）

| 层级 | 内容 |
|------|------|
| L1 | Transport: WebSocket + TLS |
| L2 | Message Framing: typed envelope |
| L3 | Semantic Types: 标准消息类型库 |
| L4 | Conversation State: 状态机 |
| L5 | Discovery: 分布式注册表 |
| L6 | Trust: 加密身份 + 信任模型 |
| L7 | Error Handling: 标准错误码 |
| L8 | Context Management: 分布式上下文 |

## 三、我们的解决方案（实践）

### 核心思路：不直接对话，通过共享环境协作

```
┌─────────────────────────────────────────────────────────┐
│                    共享文件系统                          │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │大纲.md  │  │正文.md  │  │审校.md  │  │版本记录│ │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘ │
└─────────────────────────────────────────────────────────┘
         ↑              ↑              ↑
         │              │              │
      富贵(写)      旺财(写)       Anna(审)
```

---

## 四、五大实践要点

### 1. 环境沟通 (Stigmergy)

**原理**：不直接发消息，而是通过修改共享环境来传递信息

**实践**：
- 富贵更新 `Outline.md` → 旺财自己去读
- 不发几万字的消息 → 发一个文件路径
- 用文件系统作为协议

---

### 2. 状态锁与控制权

**问题**：多Agent操作同一文件会冲突

**实践**：
| 阶段 | 锁定者 | 权限 |
|------|--------|------|
| 初稿 | 旺财 | 写，其他只能读 |
| 审校 | Anna | 审核，旺财不能改 |
| 定稿 | 富贵 | 最终确认 |

**交接指令格式**：
```
[HANDOFF: 旺财 -> Anna]
[STATUS: 审校中]
[CHECKLIST: 通过]
```

---

### 3. 可中断性与纠偏

**问题**：LLM可能陷入幻觉，不能等写完才报错

**实践**：
- 分块提交：每写完一场戏就推送
- 外部打断：利用 `subagents steer/kill` 随时干预
- 定期同步：每小时检查进度

---

### 4. 强制验证 (代替虚假共识)

**问题**：Agent可能"假装理解"

**实践**：
- 不用"收到"作为确认
- 必须通过 Checklist 验证
- 结构化核对：逻辑、设定、文笔

**Checklist 格式**：
```markdown
## 审校清单
- [ ] 逻辑通顺
- [ ] 人设一致
- [ ] 无违禁词
- [ ] 字数达标
```

---

### 5. 信息密度 (Token经济)

**问题**：废话就是在烧钱

**实践**：
- 反馈必须结构化：
  ```
  【行动点】删除第三段
  【原因】违背"展现大于讲述"
  【优先级】P1
  ```
- 拒绝"我觉得不太对"
- 用状态码代替礼貌废话

---

## 五、通信协议设计

### 消息格式

```json
{
  "type": "handoff|status|feedback|checklist",
  "from": "agent-id",
  "to": "agent-id",
  "action": "write|review|approve",
  "payload": {
    "file": "path/to/file",
    "summary": "简要说明"
  },
  "timestamp": "2026-03-17T12:00:00Z"
}
```

### 状态流转

```
[初稿] --(旺财完成)--> [待审校] --(Anna通过)--> [定稿]
   ^                          |
   |                          |
   +--------(打回修改)---------+
```
