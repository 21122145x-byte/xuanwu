# Writer-Agent 第 1 周执行清单

## 周目标

第 1 周不追求写出好作品，只做一件事：

**把 Writer-Agent 的最小系统搭起来。**

本周重点：
- 建任务类型
- 建 Memory 分层
- 建基础模板
- 建自审与反馈机制
- 跑一轮最小闭环

本周交付：
- Writer-Agent SOP
- Memory Schema
- 基础模板 4 件套
- MVP 流程

---

## Day 1：任务类型与工作流草案

### 目标
明确系统到底要处理哪些任务，以及统一动作顺序。

### 动作
1. 定义任务类型：
   - `research`
   - `outline`
   - `draft`
   - `revision`
   - `self_review`
2. 写出统一工作流草案：
   - 任务识别
   - 调记忆
   - 生成约束卡
   - 先出骨架
   - 再写正文
   - 自审
   - 回写
3. 明确每个任务类型的输入 / 输出

### 当日产出
- 任务类型表
- 工作流草案 v1

### 验收标准
- 每种任务都能说清输入 / 输出
- 工作流不少步骤，也不乱加步骤

---

## Day 2：梳理现有 Memory 来源

### 目标
搞清楚已有记忆在哪里，不再靠脑补。

### 动作
1. 列出 Identity 来源：
   - `SOUL.md`
   - `USER.md`
   - `MEMORY.md`
2. 列出 Project 来源：
   - `knowledge-base/01-Projects/...`
3. 列出 Daily / Runtime 来源：
   - `memory/YYYY-MM-DD.md`
   - 当前任务 / handoff
4. 列出 Learning 来源：
   - Feedback
   - Revisions
   - Checklist
   - 盲测记录
5. 标出哪些是高价值，哪些是噪音

### 当日产出
- Memory 来源清单
- 高价值记忆源列表

### 验收标准
- 能说清四类记忆分别从哪里来
- 至少识别出 3 类噪音源 / 待清理源

---

## Day 3：建立 Memory 分层

### 目标
把记忆按功能分层，不再混成一锅。

### 动作
1. 定义四层：
   - Identity
   - Project
   - Task
   - Learning
2. 给每层写清：
   - 存什么
   - 不存什么
   - 生命周期
   - 触发场景
3. 先手工整理首批候选记忆

### 当日产出
- Memory 分层说明
- 首批候选记忆清单

### 验收标准
- 每层边界清楚
- 不再把项目事实写进长期原则里
- 不再把临时任务混进项目记忆里

---

## Day 4：建立基础模板

### 目标
把工作流里的关键卡片定下来。

### 动作
建立以下模板：
1. 任务卡
2. 约束卡
3. 骨架卡
4. 反馈卡

每张卡都要明确字段，不要空话。

### 当日产出
- 模板 4 件套 v1

### 验收标准
- 每张卡都能直接用
- 模板字段少而够用
- 不出现“自由发挥型模板”

---

## Day 5：建立自审与基础 Checklist

### 目标
让系统开始具备最基本的 Judgment。

### 动作
1. 固定“自审五问”
2. 定义 4 类错误：
   - 结构错误
   - 人物错误
   - 节奏错误
   - 语言错误
3. 写出基础 Checklist
4. 明确每类错误的优先级判断方式

### 当日产出
- 自审五问卡
- 错误分类表
- 基础 Checklist

### 验收标准
- 每条错误都能归类
- 自审问题能直接作用于删改

---

## Day 6：跑一轮最小闭环

### 目标
用一个小任务验证整套系统能不能跑。

### 动作
找一个小任务，完整跑：
1. 识别任务类型
2. 调用相关记忆
3. 生成约束卡
4. 先做骨架
5. 写一小段正文
6. 跑自审
7. 产出反馈
8. 回写 1 条 learning

### 当日产出
- 一轮闭环记录
- 1 段样本文本
- 1 条 learning memory

### 验收标准
- 全链路跑通
- 至少发现 1 个流程问题
- 至少沉淀 1 条可复用规则

---

## Day 7：复盘与修系统

### 目标
不继续扩内容，先修系统。

### 动作
1. 看哪一步最卡
2. 看哪张模板最没用
3. 看记忆分层是否混乱
4. 看流程是否太长或太虚
5. 修一次 SOP / 模板 / 分层定义

### 当日产出
- 第 1 周复盘卡
- 修订后的流程 v1.1
- 下周重点清单

### 验收标准
- 能明确指出 3 个最卡点
- 至少修掉 1 个明显冗余环节
- 为第 2 周记忆接入做好准备

---

## 本周最低成功标准

只要做到以下 4 条，第 1 周就算过关：

1. 写作前不再直接冲正文
2. 记忆开始分层
3. 反馈开始结构化
4. 至少跑通 1 次完整闭环

---

## 一句话原则

**第 1 周不要追求“写得多好”，只追求“系统先活起来”。**
