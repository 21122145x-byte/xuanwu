# Writer-Agent + MemOS 接入设计 v1

## 目标

把 MemOS 接成 Writer-Agent 的记忆操作层，实现：
- 写前自动召回相关记忆
- 写中按固定流程推进
- 写后把反馈回写为可复用规则
- 长项目中减少失忆、串线、重复犯错

---

## 一、接入原则

1. **不推翻现有文件系统**
   - 保留现有 Markdown / Knowledge Base 作为事实源
   - MemOS 做索引、检索、回写，不先做全量替代

2. **先做最小闭环**
   - 写前检索
   - 写后回写
   - 项目切换可正确召回

3. **先服务写作任务，不追求全能记忆**
   - 只围绕 Writer-Agent 场景设计
   - 优先支持：大纲、章节初稿、改稿、审校前自查

---

## 二、四层记忆结构

### 1. Identity Layer
来源：
- `SOUL.md`
- `USER.md`
- `MEMORY.md`

存储内容：
- Agent 身份
- 用户偏好
- 稳定审美标准
- 创作禁忌
- 长期有效方法论

### 2. Project Memory Layer
来源：
- `knowledge-base/01-Projects/...`
- 世界观 / 角色 / 章节 / 协作 / 反馈文件

存储内容：
- 项目世界观
- 角色边界
- 已确认结构
- 当前阶段状态
- 已废弃方案与原因

### 3. Task Runtime Layer
来源：
- 当前任务输入
- 当前轮约束
- 当前交付要求

存储内容：
- 当前任务类型
- 本轮重点问题
- 本次禁忌
- 交付物要求

### 4. Learning Layer
来源：
- 反馈记录
- Revisions
- Checklist
- 盲测结果

存储内容：
- 错误类型
- 修正原则
- 有效写法
- 可复用经验

---

## 三、文件映射动作

### 动作 1：建立四类索引源

#### Identity Source
- 读取：`SOUL.md` `USER.md` `MEMORY.md`
- 抽取：偏好、原则、禁忌、创作定位

#### Project Source
- 读取：项目目录下 `Worldbuilding` `Characters` `Chapters` `Feedback` `Revisions`
- 抽取：设定事实、角色规则、结构状态、协作状态

#### Runtime Source
- 读取：当前用户任务 / 当前 handoff / 当前修订要求
- 抽取：本轮任务目标与约束

#### Learning Source
- 读取：Checklist、盲测、历史反馈
- 抽取：错误 -> 原因 -> 修正原则

## 四、Writer-Agent 工作流 SOP

## SOP-01：新写作任务启动

### 输入
- 用户任务 / handoff / 项目状态

### 动作
1. 识别任务类型：
   - `research`
   - `outline`
   - `draft`
   - `revision`
   - `self_review`
2. 从 MemOS 检索：
   - Identity bundle
   - Project bundle
   - 最近 Learning bundle
3. 生成一张“创作约束卡”

### 输出
创作约束卡必须包含：
- 本次任务
- 本次目标
- 本次重点
- 本次禁忌
- 必守边界
- 成功标准

---

## SOP-02：正文生成前

### 动作
1. 先输出骨架，不直接写正文：
   - 一句话梗概
   - 分段功能
   - 冲突推进
   - 信息释放顺序
   - 结尾钩子
2. 用骨架反查约束卡
3. 不符合则先修骨架

### 通过条件
- 人物目标清楚
- 段落功能清楚
- 没有明显设定堆砌
- 结尾有拉力

---

## SOP-03：正文生成

### 动作
1. 根据骨架写正文
2. 写作时优先遵守：
   - 人物先于设定
   - 冲突先于说明
   - 动作/身体感先于抽象判断
3. 禁止一边写一边扩设定

### 输出
- 正文草稿
- 简版写作备注（本轮是否偏离）

---

## SOP-04：写后自审

### 动作
固定回答 5 问：
1. 这段真正推进了什么？
2. 最有力的点是什么？
3. 最虚的地方是什么？
4. 哪句最像作者在用力？
5. 如果删 40%，先删哪里？

### 输出
- 自审结论
- 是否进入下一阶段

---

## SOP-05：反馈回写

### 输入
- 人类反馈 / Anna 审校 / 盲测结果 / 自审结论

### 动作
每条有效反馈都转换成：
1. 问题定位
2. 错误类型
3. 修正原则
4. 记忆去向：
   - `identity`
   - `project`
   - `learning`
   - `temporary`

### 输出
- 新增 Memory Object
- 更新相关项目状态

---

## SOP-06：项目切换

### 动作
1. 切换项目时先清空 Runtime Layer
2. 重新加载目标项目的 Project bundle
3. 仅保留 Identity Layer + 通用 Learning Layer
4. 禁止把上一项目的临时记忆带入新项目

### 目的
- 防止串设定
- 防止错用旧约束

---

## 五、最小可行版本（MVP）

### 第 1 周只做 4 件事
1. 建立 Identity / Project / Learning 三类映射
2. 每次写前固定检索三类 bundle
3. 每次改稿后固定回写 1 条修正原则
4. 每次项目切换时强制重载项目记忆

### MVP 验收标准
- 同类错误重复率下降
- 长项目串线减少
- 改稿说明更具体
- 开头 / 角色出场稳定性提升

---

## 六、执行优先级

### P0
- Identity 映射
- Project 映射
- 写前检索
- 写后回写

### P1
- 任务类型化检索
- 反馈分类化
- 项目切换机制

### P2
- 记忆权重
- 生命周期管理
- 自动化衰减

---

## 七、一句话执行令

**先不重构所有文件；先把 MemOS 接成“写前会调、写后会记、项目切换不串线”的 Writer-Agent 记忆层。**
