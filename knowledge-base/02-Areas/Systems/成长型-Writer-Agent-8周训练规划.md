# 成长型 Writer-Agent 8 周训练规划

## 目标

把普通 agent 训练成具备持续成长能力的 Writer-Agent。

核心不只是“会写”，而是同时具备：
- `Research`
- `Outline`
- `Writing`
- `Judgment`
- `Memory`
- `Workflow`
- `Feedback`
- `Evaluation`

最终目标：
- 写前会调记忆
- 写中按流程推进
- 写后会回写经验
- 长项目中不易失忆、串线、重复犯错

---

## 一、总周期

总周期：**8 周**

理由：
1. 先打底
2. 再建流程
3. 再跑闭环
4. 最后验证是否真正成长

---

## 二、四阶段

## 阶段 1：基础搭建（第 1–2 周）

### 目标
先解决：
- 不失忆
- 不乱写
- 不乱改

### 核心任务
- 建立 Memory 分层
- 建立 Writer-Agent 工作流
- 建立反馈模板
- 建立评估基线
- 明确任务类型

### 成果标准
- 每次写前先调记忆
- 不再直接冲正文
- 每次反馈有结构化记录
- 项目切换不串线

---

## 阶段 2：核心能力训练（第 3–4 周）

### 目标
分别强化：
- `Research`
- `Outline`
- `Writing`
- `Judgment`

### 核心任务
- Research 转叙事
- Outline 骨架搭建
- Writing 节奏与人物控制
- Judgment 自审与删改

### 成果标准
- 资料能转化
- 先骨架后正文开始固定化
- 改稿理由更具体
- 低级错误减少

---

## 阶段 3：闭环联调（第 5–6 周）

### 目标
把 Memory、Workflow、Feedback 与四项核心能力真正串起来。

### 核心任务
- 跑真实任务
- 记录错误
- 回写经验
- 下一轮验证是否改善

### 成果标准
- 形成至少一轮完整闭环
- 相似问题更早避开
- 开头 / 人物出场 / 节奏更稳

---

## 阶段 4：稳定与升级（第 7–8 周）

### 目标
从“能跑”推进到“越来越像一个作家型 agent”。

### 核心任务
- 强化长期原则
- 清理无效记忆
- 升级高价值 learning
- 看跨任务复用是否成立

### 成果标准
- 出现稳定可复用规则
- Writer-Agent 风格与判断倾向开始成形
- 不再每次都靠人工从头纠偏

---

## 三、标准周节奏

## 周一：Memory + Planning Day

动作：
1. 读 Identity / Project / Recent Learning
2. 清理过期 task memory
3. 更新本周项目状态
4. 生成本周目标卡
5. 生成今天任务卡

产出：
- 本周目标卡
- 今日任务卡
- 本周禁忌清单

---

## 周二：Research Day

动作：
1. 选 1 个主题
2. 搜集资料
3. 做三层提炼：事实 / 机制 / 戏剧
4. 提炼 2 个冲突点、2 个场景资源、1 个故事切口
5. 回写 research learning

产出：
- 主题卡片
- 三层提炼
- 1 条 learning

---

## 周三：Outline Day

动作：
1. 做一句话梗概
2. 拆章节 / 场景功能
3. 设计开头钩子 / 中段支撑 / 结尾拉力
4. 做结构自查
5. 回写结构 learning

产出：
- 3 个一句话梗概
- 1 版章节骨架
- 1 份场景功能表

---

## 周四：Writing Day

动作：
1. 先读约束卡
2. 根据骨架写正文
3. 同一场景写两种版本（可选）
4. 做语言压缩
5. 标记疑点，不当场扩设定

产出：
- 场景稿 / 章节稿
- 1 次压缩改写结果
- 写作备注

---

## 周五：Judgment Day

动作：
1. 跑自审五问
2. 局部 / 整体分离评估
3. 标注结构 / 人物 / 节奏 / 语言错误
4. 输出删改优先级
5. 回写判断 learning

产出：
- 自审卡
- 错误分类表
- 1 条修正原则

---

## 周六：Revision + Feedback Day

动作：
1. 处理外部反馈 / 自审反馈
2. 转成结构化反馈：问题定位 / 错误类型 / 修正原则
3. 改一版
4. 写回 memory
5. 更新项目状态

产出：
- 修订稿
- 结构化反馈卡
- 1–2 条 learning memory

---

## 周日：Review + Reset Day

动作：
1. 看本周重复错误
2. 看哪条规则最有效
3. 看哪些记忆该升级 / 清理
4. 看下周训练重点
5. 做阶段总结

产出：
- 周复盘卡
- 记忆清理列表
- 下周重点列表

---

## 四、8 周每周重点

## 第 1 周：系统起盘周
- 建任务类型表 + 工作流草案
- 梳理现有 Memory 来源
- 建 Memory 分层
- 建约束卡 / 骨架卡 / 反馈卡
- 建自审五问 / 基础 checklist
- 跑一次最小闭环
- 修系统，不求高产

周交付：
- Writer-Agent SOP
- Memory Schema
- 基础模板 4 件套
- MVP 流程

## 第 2 周：记忆接入周
- 清点 Identity 记忆
- 清点 Project 记忆
- 整理首批 Learning 记忆
- 建立 Task 记忆机制
- 设计检索触发规则
- 跑一次真实写作任务，强制前检索后回写
- 复盘哪些记忆有用，哪些是垃圾

周交付：
- 首批核心 Memory Objects
- 检索规则
- 回写规则

## 第 3 周：Research 强化周
- 研究模板定稿
- 主题卡片训练
- 三层提炼训练
- 练“research → 2 场景 + 1 切口”
- 评估 research 是否真能写
- 将 research 结果接到 outline
- 总结什么资料真的有用

周交付：
- Research 模板
- 3–5 个主题卡片样例
- 一批 research learning

## 第 4 周：Outline 强化周
- 一句话梗概训练
- 章节功能训练
- 钩子 / 转折点训练
- 场景功能标注训练
- 结构审查训练
- 研究 → 骨架 → 一小段正文联动
- 总结结构类常见错误

周交付：
- Outline 模板
- 骨架样例
- 结构类 learning 规则

## 第 5 周：Writing 强化周
- 人物出场训练
- 场景进入感训练
- 同一内容三写法
- 压缩删改训练
- 人物台词一致性训练
- 完整写一个场景并自审
- 总结语言与画面问题

周交付：
- Writing 约束卡
- 场景样例
- 语言类 learning 规则

## 第 6 周：Judgment 强化周
- 自审五问训练
- 局部 / 整体分离评估
- 错误分类训练
- 删改优先级训练
- 用反馈提炼修正原则
- 完整跑一轮“写→审→改”
- 总结常见误判

周交付：
- Judgment 自审卡
- 错误分类表
- 修正原则库初版

## 第 7 周：闭环联调周
- 用真实任务启动
- Research
- Outline
- Writing
- Judgment
- Revision + 回写
- 看记忆是否真的帮助下一轮

周交付：
- 一轮完整闭环记录
- 闭环问题清单
- 规则升级清单

## 第 8 周：稳定与升级周
- 清理低质量 memory
- 提升高价值 learning
- 整理跨项目可复用规则
- 压缩 workflow 冗余
- 看是否形成稳定写作倾向
- 跑一次最终验证任务
- 形成阶段总结与下一周期目标

周交付：
- 稳定版 Writer-Agent 流程
- 精炼版 Memory 结构
- 下一周期升级计划

---

## 五、每日时间分配

### Block A：同步（20–30 分钟）
- 读记忆
- 对齐任务
- 生成约束卡

### Block B：主训练 / 主生产（60–90 分钟）
- Research 或 Outline 或 Writing 主任务

### Block C：二次加工（45–60 分钟）
- 压缩、改写、补骨架、查缺口

### Block D：自审 + 回写（20–30 分钟）
- 做 Judgment
- 写回 learning
- 更新项目状态

---

## 六、核心成长指标

1. 重复错误率
2. 开头通过率
3. 结构返工率
4. 反馈转规则率
5. 跨任务复用率

---

## 七、一句话总结

**8 周周期，4 个阶段，7 天节奏，4 个日 block。用 `Research / Outline / Writing / Judgment` 训练写作核心，用 `Memory / Workflow / Feedback / Evaluation` 形成成长闭环。**
