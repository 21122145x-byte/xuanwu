# Writer-Agent 第 2 周执行清单

## 周目标

第 2 周只做一件事：

**让系统真的“写前会调，写后会记”。**

本周重点：
- 清点四层记忆
- 建立首批核心 Memory Objects
- 建立检索触发规则
- 建立回写规则
- 跑一次真实写作任务验证记忆闭环

本周交付：
- 首批核心 Memory Objects
- 检索规则
- 回写规则
- 一轮带记忆闭环的真实任务记录

---

## Day 1：清点 Identity 记忆

### 目标
把长期稳定的创作人格、用户偏好、审美原则提出来。

### 动作
1. 读取 `SOUL.md` `USER.md` `MEMORY.md`
2. 提炼：
   - 长期偏好
   - 红线
   - 稳定创作原则
3. 先只保留高价值内容

### 当日产出
- Identity Memory 候选清单

### 验收标准
- 至少 5 条候选
- 每条都可执行，不写空话

---

## Day 2：清点 Project 记忆

### 目标
把当前项目真正重要的事实、人物边界、结构状态提出来。

### 动作
1. 读取项目目录关键文件
2. 提炼：
   - 项目核心冲突
   - 角色边界
   - 当前阶段
   - 已确认方向
3. 去掉过时或未确认内容

### 当日产出
- Project Memory 候选清单

### 验收标准
- 至少 8 条候选
- 能区分项目事实与个人判断

---

## Day 3：整理首批 Learning 记忆

### 目标
把已经验证有效的修正原则从反馈中提出来。

### 动作
1. 读取 Feedback / Revisions / 盲测 / 旧修订记录
2. 提炼：
   - 常见错误
   - 对应修正原则
   - 已验证有效的方法
3. 区分：
   - 可跨项目复用
   - 只适用于当前项目

### 当日产出
- Learning Memory 候选清单

### 验收标准
- 至少 4 条候选
- 每条都包含“错误 -> 原因 -> 修正原则”

---

## Day 4：建立 Task 记忆机制

### 目标
让每次任务都有临时工作记忆，不再全靠聊天上下文。

### 动作
1. 定义 Task Memory 字段
2. 规定任务启动时必须生成 Task Memory
3. 规定任务结束时清理 temporary memory

### 当日产出
- Task Memory 规则
- 临时记忆生成 / 清理规则

### 验收标准
- 能明确什么该留、什么该清
- 不把临时任务写进长期层

---

## Day 5：设计检索触发规则

### 目标
让不同任务会调不同记忆，而不是全塞进来。

### 动作
1. 为以下任务定义触发：
   - `write_opening`
   - `outline_chapter`
   - `revise_draft`
2. 为每类任务定义优先召回顺序
3. 定义项目切换时保留 / 清空规则

### 当日产出
- Retrieval Trigger 规则表

### 验收标准
- 每类任务都有清晰召回逻辑
- 项目切换不串 project memory

---

## Day 6：跑一次真实写作任务

### 目标
验证记忆检索与回写机制不是纸上谈兵。

### 动作
找一个真实任务，完整跑：
1. 任务识别
2. 前置检索记忆
3. 生成约束卡
4. 出骨架
5. 写正文
6. 自审
7. 写回 1 条 feedback / learning

### 当日产出
- 一轮真实任务记录
- 1 次前检索结果
- 1 条新增记忆

### 验收标准
- 记忆确实参与了写作前准备
- 至少有 1 条经验被成功回写

---

## Day 7：复盘哪些记忆有用，哪些是垃圾

### 目标
开始做记忆质量控制，不让系统越堆越脏。

### 动作
1. 复盘本周所有候选记忆
2. 删除 / 降级低质量项
3. 提升高价值项优先级
4. 修一次检索规则

### 当日产出
- 第 2 周复盘卡
- 精炼版首批 Memory Objects
- 修订后的检索规则

### 验收标准
- 至少识别 3 条低质量记忆
- 至少保留 10 条高价值记忆
- 检索规则更清晰

---

## 本周最低成功标准

只要做到以下 4 条，第 2 周就算过关：

1. 建出首批核心 Memory Objects
2. 不同任务开始调不同记忆
3. 至少跑通 1 次真实任务的前检索 + 后回写
4. 开始有记忆清理意识

---

## 一句话原则

**第 2 周不是堆更多记忆，而是让记忆真正参与工作。**
