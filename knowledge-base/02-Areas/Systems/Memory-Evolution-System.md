## 一、五层记忆架构 (Memory Architect)

### 核心思想
记忆不是单一文件，而是分层、分级、分频率的持久化系统。每层解决不同问题，组合实现高可靠性。

### 五层结构

```
┌─────────────────────────────────────────────────────────────┐
│                    五层记忆架构                               │
├─────────────────────────────────────────────────────────────┤
│  L1: Daily Flush      (上下文感知，自动触发)                   │
│  L2: Hourly Micro-Sync (每日4次，健康检查)                    │
│  L3: Daily Context Sync (23:30，全量对账)                     │
│  L4: Weekly Compound  (周日22:00，知识蒸馏)                   │
│  L5: Disaster Recovery (备份、恢复、损坏检测)                 │
└─────────────────────────────────────────────────────────────┘
```

#### L1: Daily Flush (日刷)
**触发条件：**
- 上下文窗口 > 80%
- 会话结束（心跳超时30分钟）
- 关键操作完成
- 手动触发 `/flush`

**刷入内容：**
- 关键决策（带理由）
- 已完成的行动
- 新学到的知识
- TODO更新

**存储位置：** `memory/YYYY-MM-DD.md`

#### L2: Hourly Micro-Sync (微同步)
**时间表：**
| 时间 | 检查项 | 目的 |
|------|--------|------|
| 00:00 | 明日文件预创建 | 防止遗漏 |
| 09:00 | 晨间同步 | 准备工作 |
| 16:00 | 午后检查 | 健康监控 |
| 20:00 | 晚间同步 | 工作备份 |

#### L3: Daily Context Sync (日对账)
**时间：** 每日 23:30

**流程：**
1. 收集当日所有会话片段
2. 重建时间线
3. 识别上下文断层
4. 从日志填补缺口
5. 生成日总结
6. 重要事件标记周汇总

#### L4: Weekly Compound (周蒸馏)
**时间：** 周日 22:00

**蒸馏标准：**
| 类型 | 标准 | 动作 |
|------|------|------|
| 决策 | 提及≥2次或标记重要 | 写入 MEMORY.md |
| 学习 | 有证据的新洞见 | 写入 Lessons Learned |
| 偏好 | 连续3次一致模式 | 更新 User Profile |
| 项目 | 活跃且有进展 | 更新 Active Projects |

#### L5: Disaster Recovery (灾备)
**备份层级：**
- T1: 实时 WAL（每次写入）
- T2: 每小时快照
- T3: 每日快照 + Git同步
- T4: 每周归档 + 完整性校验

**恢复优先级：**
1. WAL恢复
2. 每小时快照
3. 每日快照
4. Git历史
5. 会话日志重建

## 二、自我进化系统 (Self Evolver)

### 核心思想
> "每次失败都是学习机会。每个模式都是优化目标。"

### 进化循环

```
┌─────────────────────────────────────────┐
│           自我进化循环                    │
├─────────────────────────────────────────┤
│                                          │
│  1. 收集日志 ←────┐                      │
│     ↓             │                      │
│  2. 分析失败      │                      │
│     ↓             │                      │
│  3. 识别模式      │                      │
│     ↓             │                      │
│  4. 优化策略      │                      │
│     ↓             │                      │
│  5. 测试验证 ────┘                       │
│     ↓                                    │
│  6. 部署上线                             │
│     ↓                                    │
│  7. 监控效果 ──→ 回到1                   │
│                                          │
└─────────────────────────────────────────┘
```

### 分析维度

#### 1. 失败日志分析
```python
分析维度 = {
    'api_errors': 'API错误频率',
    'timeout_errors': '超时模式',
    'logic_errors': '逻辑错误',
    'context_errors': '上下文错误'
}
```

#### 2. 模式识别
| 模式 | 检测条件 | 优化动作 |
|------|----------|----------|
| 重复失败 | 同错误>3次 | 更新错误处理策略 |
| 慢操作 | 操作>30秒 | 实现缓存或异步 |
| 上下文溢出 | 频繁>80% | 优化提示压缩 |
| API限流 | 限流错误 | 实现退避策略 |

#### 3. 优化领域
- **提示工程**: 减少截断率 < 5%
- **错误处理**: 恢复成功率 > 90%
- **上下文管理**: 效率提升 +30%
- **工具选择**: 准确率 > 95%
- **响应质量**: 用户满意度 > 90%

### 安全边界
1. **绝不自动应用关键变更**
   - API密钥、认证令牌
   - 核心配置
   - 破坏性操作

2. **部署前必须测试**
   - 单元测试
   - A/B测试
   - 回滚能力

3. **重大变更需人工批准**
   - 架构变更
   - 新技能创建
   - 策略修改

## 三、知识库流程 (Knowledge Bridge)

### 核心思想
记忆是临时的，知识是持久的。自动将记忆转化为结构化知识，实现"第二大脑"。

### 架构

```
OpenClaw Memory (运行时)
    ↓ (自动同步)
Knowledge Base (持久化)
    ↓ (查询)
Long-term Context (长期上下文)
```

### 目录结构

```
knowledge-base/
├── 00-Inbox/           # 收件箱（临时）
├── 01-Projects/        # 项目
├── 02-Areas/           # 知识领域
├── 03-Resources/       # 资源
├── 04-Archive/         # 归档
├── 30-evergreen/       # 永久笔记
└── 99-Meta/            # 索引
```

### 自动分类规则

| 内容模式 | 分类 | 目录 |
|----------|------|------|
| 包含"项目"、"Project" | 项目 | 01-Projects/ |
| 日期格式 | 日志 | Daily/ |
| 人名 | 人物 | People/ |
| 技能、技术 | 知识 | Knowledge/ |
| 待办、TODO | 任务 | Tasks/ |

### 同步流程

#### 日同步 (23:00)
```bash
python3 kb_sync.py today
```
- 将当日 memory 同步到知识库
- 自动分类归档
- 更新索引

#### 双向同步
```python
# Memory → KB (自动)
knowledge.sync_to_vault(
    source="memory/2026-03-11.md",
    target="Daily/2026-03-11.md"
)

# KB → Memory (按需)
context = knowledge.sync_from_vault(
    query="Projects/OpenClaw"
)
```

## 五、使用原则

### 对子 Agent 的要求

1. **记忆优先**
   - 重要决策立即写入 memory
   - 不要依赖" mental notes"

2. **分层使用**
   - 短期：memory/YYYY-MM-DD.md
   - 中期：MEMORY.md
   - 长期：knowledge-base/

3. **持续进化**
   - 记录失败和教训
   - 参与模式识别
   - 应用优化策略

4. **知识沉淀**
   - 项目结束归档到知识库
   - 建立双向链接
   - 维护索引更新

### 关键文件位置

```
~/.openclaw/workspace/knowledge-base/     # 公用知识库
~/.openclaw/workspace-fugui/memory/       # 富贵记忆
~/.openclaw/workspace-fugui/MEMORY.md     # 富贵长期记忆
```
