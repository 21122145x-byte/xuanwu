## 🔄 与其他系统的集成

### 与 skill-vetter 的集成

```python
# skill-manager 调用 skill-vetter 进行审计
class SkillManager:
    def install_skill(self, source):
        # 1. 下载 skill
        skill_path = self.download(source)
        
        # 2. 调用 vetter 审计
        audit_result = self.vetter.audit(skill_path)
        
        if not audit_result.passed:
            raise SecurityError(audit_result.issues)
        
        # 3. 继续安装
        self.register(skill_path)
```

### 与 skill-finder 的集成

```python
# skill-manager 使用 skill-finder 搜索
class SkillManager:
    def search_skills(self, keyword):
        # 1. 本地搜索
        local_results = self.registry.search(keyword)
        
        # 2. 调用 finder 智能推荐
        recommended = self.finder.recommend(keyword)
        
        # 3. 合并结果
        return self.merge_results(local_results, recommended)
```

### 与 github-automation 的集成

```python
# skill-manager 触发 github 备份
class SkillManager:
    def after_install(self, skill_name):
        # 安装后触发备份
        self.github.backup_skill(skill_name)
        
    def after_update(self, skill_name):
        # 更新后触发备份
        self.github.commit_changes(f"Updated {skill_name}")
```

## 📝 配置管理

```yaml
# skill-manager 配置
skill_manager:
  # 权限配置
  permissions:
    auto_install_trusted: true  # 自动安装可信来源
    auto_update_security: true  # 自动更新安全补丁
    require_approval_for_major: true  # 大版本需批准
  
  # 集成配置
  integrations:
    vetter:
      enabled: true
      auto_audit: true
    finder:
      enabled: true
      suggest_threshold: 0.7
    github:
      enabled: true
      auto_backup: true
  
  # 工作流配置
  workflows:
    pre_install:
      - check_conflicts
      - audit_security
      - backup_current
    post_install:
      - update_registry
      - update_index
      - trigger_backup
    pre_update:
      - backup_current
      - check_compatibility
    post_update:
      - audit_security
      - update_registry
      - trigger_backup
```

---

这个设计让 skill-manager 成为 skill 生态系统的"管理中心"，协调各个子系统，确保 skill 的安全、有序管理。
